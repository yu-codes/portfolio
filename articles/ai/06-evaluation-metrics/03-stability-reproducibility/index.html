穩定性與可重現性

確保實驗結果可靠：統計顯著性檢定與實驗可重現性的實務方法

你改了一個超參數，模型效能從 85.2% 提升到 86.1%。這個改善是真實的嗎？還是只是隨機波動？穩定性和可重現性是科學化做 ML 的基石。

隨機性的來源

ML 實驗中的隨機性來自很多地方：

  • 資料分割：不同的 train-test split 會得到不同結果
  • 權重初始化：不同的隨機初始化導致不同的收斂點
  • 資料順序：訓練資料的打亂順序影響學習路徑
  • Dropout：每次遮蔽不同的神經元
  • GPU 非確定性：某些 CUDA 操作結果不完全確定

統計顯著性檢定

為什麼需要？

兩個模型在同一個測試集上的效能差距，可能只是隨機波動。統計檢定幫你判斷「這個差距是否足夠大，不太可能是偶然」。

常用方法

配對 t 檢定：在交叉驗證的多折結果上，比較兩個模型的平均效能是否有顯著差異。

McNemar 檢定：專為分類器比較設計。看兩個模型「不一致」的預測：一個對另一個錯的次數，是否顯著不對稱。

Bootstrap 信賴區間：從測試集中重複抽樣，計算指標的分佈。如果兩個模型的信賴區間不重疊，差異就是顯著的。

多重比較問題

如果你同時比較了 10 個模型，其中一個「碰巧」表現最好的機率很高。需要用 Bonferroni 校正或 Holm-Bonferroni 方法調整 p 值。

可重現性的實踐

固定隨機種子

這是最基本的步驟。需要同時固定：

  • Python 的 random 種子
  • NumPy 的隨機種子
  • 框架（PyTorch/TensorFlow）的隨機種子
  • CUDA 的隨機種子

但要注意：固定種子不等於結果完全一致。某些 GPU 操作天然有非確定性。

實驗記錄

每次實驗都要記錄：

  • 程式碼版本（Git commit hash）
  • 資料版本
  • 完整的超參數配置
  • 環境資訊（GPU 型號、CUDA 版本、Python 版本）
  • 所有結果指標

消融實驗（Ablation Study）

驗證每個改動的貢獻：每次只改一個因素，觀察效能變化。

例如，你的模型有三個改進：新的資料增強、新的損失函數、新的學習率排程。消融實驗就是分別移除每一個，看效能下降多少。

這能幫你理解每個改進的真實貢獻，避免「把不相關的改動歸功於某個特定改進」。

結果報告的最佳實踐

  • 報告均值 ± 標準差（至少跑 3-5 次不同種子）
  • 報告多個指標，不要只挑最好的
  • 明確描述實驗設定（資料分割方式、超參數搜尋範圍）
  • 提供程式碼和資料以供復現

實務心法

  1. 效能差距 < 1% 時，先做統計檢定再下結論
  2. 固定隨機種子是最低要求——但不要只報告最好種子的結果
  3. 用實驗追蹤工具——手動記錄遲早會出錯
  4. 消融實驗是驗證假設的唯一可靠方法
