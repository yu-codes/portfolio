現實世界的分類問題很少是完美平衡的二元分類。多類別分類和類別不平衡是你必須面對的常態。

多類別分類的指標

從二元到多類別

二元分類的 Precision、Recall 可以直接擴展到多類別。關鍵在於如何「平均」各類別的指標。

Micro 平均：把所有類別的 TP、FP、FN 加總後計算。適合重視整體效能。

Macro 平均：先算每個類別的指標，再取平均。對每個類別同等重視，不管它的樣本數多少。

Weighted 平均：像 Macro，但用各類別的樣本數作為權重。

如何選擇？

  • 每個類別都重要 → Macro
  • 整體效能最重要 → Micro（等同 Accuracy）
  • 想反映類別大小的差異 → Weighted

混淆矩陣（多類別版）

多類別混淆矩陣是 N×N 的矩陣，能直觀看到哪些類別容易被混淆。

常見發現：

  • 某些類別總是被誤判為另一個特定類別 → 可能特徵太相似
  • 某個類別的 Recall 極低 → 可能樣本太少

類別不平衡問題

類別不平衡是最普遍的實務問題之一：

  • 欺詐偵測：0.1% 是欺詐
  • 疾病診斷：1% 是罕見疾病
  • 異常偵測：>99% 是正常

不平衡的影響

模型傾向於預測多數類——這樣就能「輕鬆」獲得高準確率，但完全無法識別少數類。

資料層面的解決方案

過採樣（Oversampling）：增加少數類的樣本數。

  • 隨機複製：最簡單但可能造成過擬合
  • SMOTE：在少數類樣本之間插值，生成新的合成樣本
  • ADASYN：在決策邊界附近生成更多合成樣本

欠採樣（Undersampling）：減少多數類的樣本數。

  • 隨機移除：簡單但可能丟失重要資訊
  • Tomek Links：移除距離少數類最近的多數類樣本
  • 聚類後欠採樣：保留多數類的代表性樣本

組合方法：同時使用過採樣和欠採樣。

演算法層面的解決方案

類別權重：在損失函數中給少數類更高的權重。

Focal Loss：自動降低簡單樣本（通常是多數類）的權重，讓模型聚焦在困難樣本上。

閾值調整：降低少數類的預測閾值，讓它更容易被預測到。

多標籤分類

一個樣本可以同時屬於多個類別（如一篇文章可以同時標記為「科技」和「商業」）。

特殊指標

  • Hamming Loss：平均每個標籤被錯誤預測的比例
  • Subset Accuracy：所有標籤都完全正確的比例（通常很低）
  • Example-based F1：對每個樣本算 F1，再取平均

處理策略

  • Binary Relevance：對每個標籤獨立訓練一個二元分類器
  • Classifier Chains：考慮標籤間的相關性
  • Multi-Label Neural Network：最後一層用 Sigmoid（而非 Softmax），每個輸出獨立

實務心法

  1. 類別不平衡時，先調整評估指標，再考慮資料重採樣
  2. 過採樣只能在訓練集上做——驗證集和測試集必須保持原始分佈
  3. Focal Loss 通常比 SMOTE 更有效且更簡單
  4. 不要追求少數類的 100% Recall——代價是大量 False Positive
