分散式訓練的概念不難，但自己實作所有的通訊和同步邏輯非常複雜。分散式訓練框架幫你處理了這些底層細節。

PyTorch DDP（Distributed Data Parallel）

PyTorch 內建的分散式資料平行方案，是目前最基礎也最廣泛使用的選擇。

核心特色

  • 使用 NCCL 後端進行 GPU 間通訊
  • Ring AllReduce 同步梯度
  • 梯度同步與反向傳播重疊（通訊隱藏）
  • 支援多機多卡

適用場景

模型能放進單個 GPU，但想用多個 GPU 加速訓練。這涵蓋了大部分場景。

Horovod

Uber 開發的分散式訓練框架。設計目標是「讓分散式訓練像本地訓練一樣簡單」。

特色

  • 支援 TensorFlow、PyTorch、Keras
  • 使用 MPI 和 NCCL 進行通訊
  • 程式碼改動最小——只需加幾行程式碼
  • 彈性訓練支援——可以動態增減 worker

Horovod vs DDP

面向HorovodDDP
框架支援TF + PyTorch僅 PyTorch
啟動方式horovodrun / mpiruntorchrun
通訊MPI + NCCLNCCL
彈性訓練支援有限支援

目前趨勢：PyTorch 生態系統傾向使用 DDP，Horovod 在 TF 生態系統中仍有一席之地。

DeepSpeed

微軟開發的深度學習優化庫。專為訓練超大模型設計。

ZeRO 優化器

DeepSpeed 的殺手級功能。三個階段逐步減少記憶體冗餘：

  • ZeRO-1：分片優化器狀態 → 記憶體減少 4 倍
  • ZeRO-2：再分片梯度 → 記憶體減少 8 倍
  • ZeRO-3：再分片參數 → 理論上無限制模型大小

ZeRO-Offload

當 GPU 記憶體不夠時，把部分計算和資料卸載到 CPU。用 CPU 記憶體彌補 GPU 記憶體的不足。

ZeRO-Infinity

進一步卸載到 NVMe SSD。讓即使只有一張消費級 GPU 也能訓練超大模型（雖然很慢）。

其他功能

  • 混合精度訓練：自動管理 FP16/BF16 計算
  • 梯度累積：自動處理微批次
  • 稀疏注意力：降低 Transformer 的記憶體消耗
  • 管線平行：內建管線平行支援

適用場景

  • 訓練大型語言模型（>1B 參數）
  • GPU 記憶體不足以容納完整模型
  • 需要極致的訓練效率

FSDP（Fully Sharded Data Parallelism）

PyTorch 原生的 ZeRO-3 實作。

優勢

  • PyTorch 原生，不需要額外依賴
  • 與 PyTorch 生態系統完美整合
  • 活躍的社群支援

FSDP vs DeepSpeed

面向FSDPDeepSpeed
生態系統PyTorch 原生獨立庫
功能豐富度核心功能非常豐富
學習曲線較低較高
CPU Offload支援支援（更成熟）
社群PyTorch 社群微軟 + 開源社群

框架選擇建議

場景推薦
一般多 GPU 訓練PyTorch DDP
中型模型（1-10B）FSDP
超大型模型（>10B）DeepSpeed
TensorFlow 分散式訓練Horovod
HuggingFace 生態系統Accelerate（底層用 DDP/FSDP/DeepSpeed）

實務心法

  1. 從 DDP 開始——大多數場景它就夠了
  2. 模型放不進單 GPU 時，先試 FSDP——比 DeepSpeed 簡單
  3. 需要訓練超大模型時再用 DeepSpeed——它的功能最全但也最複雜
  4. HuggingFace Accelerate 是很好的抽象層——讓你輕鬆切換底層框架
