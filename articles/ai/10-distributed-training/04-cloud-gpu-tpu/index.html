不是每個團隊都有自己的 GPU 叢集。雲端 GPU/TPU 讓你可以按需使用強大的計算資源，但如何選擇和優化，直接影響你的成本和效率。

GPU 實例選擇

主流雲端 GPU

GPU記憶體特色雲端平台
T416GB入門級，推論首選AWS, GCP, Azure
A10G24GB中階，訓練與推論兼顧AWS
V10016/32GB經典，性價比高各平台
A10040/80GB旗艦，大模型訓練各平台
H10080GB最新旗艦，LLM 訓練首選各平台

如何選擇？

推論部署 → T4（便宜且足夠） 一般訓練 → A10G 或 V100（性價比最佳） 大模型訓練 → A100 或 H100（記憶體和算力的極致）

TPU（Tensor Processing Unit）

Google 自研的 AI 專用晶片，只在 Google Cloud 上提供。

TPU 的特色

  • 專為矩陣運算設計，在特定工作負載上效能極高
  • 原生支援 BF16
  • 高頻寬的 TPU 互聯，適合大規模分散式訓練
  • 定價有時比同等 GPU 更有競爭力

TPU 的限制

  • 只在 GCP 上可用
  • 主要支援 TensorFlow 和 JAX（PyTorch 支援有限，透過 XLA）
  • 某些操作可能不支援或效能不佳
  • 除錯和 Profiling 比 GPU 困難

TPU 版本

版本特色
TPU v4目前主流，用於訓練大模型
TPU v5e成本優化版本
TPU v5p效能旗艦

成本優化

Spot / Preemptible 實例

使用雲端的閒置容量，價格通常只有隨需的 60-90% 折扣。

代價：可能隨時被搶佔（中斷）。

應對策略：

  • 定期儲存 checkpoint
  • 設計可恢復的訓練流程
  • 使用多可用區以降低同時被中斷的機率

Reserved 實例

承諾使用一段時間（1-3 年），換取折扣。適合長期穩定的工作負載。

訓練效率

最好的省錢方法是減少訓練時間：

  • 混合精度訓練
  • 高效的 DataLoader
  • 避免不必要的計算
  • 使用更好的學習率排程加速收斂

自動擴縮

使用 Kubernetes + 自動擴縮，讓叢集規模根據工作負載自動調整。空閒時縮減到零。

訓練平台服務

平台特色
AWS SageMaker完整的 ML 平台，支援訓練、部署、監控
GCP Vertex AIGoogle 的 ML 平台，原生 TPU 支援
Azure ML微軟的 ML 平台，與 Azure 服務整合

這些平台提供了訓練管理、超參數調優、模型部署等功能，省去了自己管理基礎設施的麻煩。

代價：比自己管理 VM 貴 20-50%，但節省了大量的運維時間。

混合雲策略

大型組織通常採用混合策略：

  • 日常訓練：用自有的 GPU 叢集（成本最低）
  • 尖峰需求：臨時使用雲端 GPU（彈性）
  • 實驗探索：用 Spot 實例（成本最低）

實務心法

  1. Spot 實例 + 自動 checkpoint 是訓練的最佳性價比組合
  2. 先在小規模（1-2 GPU）上驗證程式碼，再擴展到大叢集
  3. TPU 適合 JAX/TF 用戶——PyTorch 用戶堅持 GPU 就好
  4. 監控你的雲端花費——設定預算告警，避免意外帳單
