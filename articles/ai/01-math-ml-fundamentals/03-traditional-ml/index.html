深度學習再怎麼火，傳統機器學習模型依然是每個 ML 工程師必須掌握的基礎。它們不僅在許多場景下表現優異（尤其是結構化資料），更重要的是，理解這些模型能幫你建立對整個 ML 領域的直覺。

線性模型：一切的起點

線性回歸

最簡單也最基礎的模型。它假設輸出與輸入之間存在線性關係。

核心概念：找到一條（或一個超平面），讓預測值與真實值的差距最小。用最小平方法可以直接算出解析解，不需要梯度下降。

線性回歸的價值不在於它的預測能力，而在於它的可解釋性——每個特徵的係數直接告訴你該特徵對結果的影響方向與大小。

邏輯回歸

雖然名字有「回歸」，但它是一個分類模型。它在線性回歸的基礎上，加了一個 Sigmoid 函數，把輸出壓縮到 0 和 1 之間，解釋為機率。

邏輯回歸是二元分類的基石，也是神經網路最後一層（Sigmoid 輸出）的原型。

決策樹：像人類一樣思考

決策樹模仿人類的決策過程：根據特徵的值，一步步做出判斷。

核心概念

  • 分裂準則：如何選擇用哪個特徵來分裂？常用 Gini 不純度或資訊增益（Information Gain）
  • 剪枝：防止樹長得太深（過擬合）的策略
  • 葉節點：最終的預測值（分類為多數類別，回歸為平均值）

優缺點

優點缺點
直覺易懂，可視覺化容易過擬合
不需要特徵縮放對資料的微小變化很敏感
能處理非線性關係單棵樹效能有限

集成學習：三個臭皮匠勝過一個諸葛亮

單棵決策樹效能有限，但把很多棵樹組合起來，效能就能大幅提升。這就是集成學習的核心思想。

Random Forest（隨機森林）

訓練很多棵決策樹，每棵樹只看到部分資料和部分特徵。最終預測取所有樹的投票（分類）或平均（回歸）。

關鍵直覺：每棵樹都會犯錯，但不同的樹犯不同的錯。把大家的意見平均起來，個別錯誤就會被抵消。

Gradient Boosting（梯度提升）

與隨機森林的「平行獨立」不同，梯度提升是依序建樹：每棵新樹專注於修正前面所有樹的殘差。

  • XGBoost：業界標配，加入正則化防止過擬合，效率極高
  • LightGBM：用直方圖近似加速訓練，處理大資料集更快
  • CatBoost：天生支援類別特徵，不需要額外編碼

在結構化資料（表格資料）的競賽和實務中，梯度提升幾乎是無可爭議的王者。

支持向量機（SVM）

SVM 的目標是找到一個最大間隔超平面，把不同類別的資料分開，同時讓間隔（margin）最大化。

核心技巧

核函數（Kernel Trick）：不需要真的把資料映射到高維空間，就能在高維空間中找到分隔超平面。常見的核函數包括線性核、多項式核和 RBF（高斯）核。

SVM 在高維空間中表現優異（如文字分類），但在大資料集上訓練速度較慢，逐漸被深度學習取代。

K-近鄰（KNN）

KNN 是最簡單的「懶學習」演算法：不建立模型，而是在預測時，找到最近的 K 個鄰居，投票決定類別。

關鍵考量：

  • K 的選擇：太小容易過擬合，太大會欠擬合
  • 距離度量：歐氏距離、曼哈頓距離、餘弦距離
  • 維度災難：在高維空間中，所有點的距離趨於相同，KNN 會失效

非監督學習

K-Means 聚類

把資料分成 K 群，讓每群的資料點盡量靠近群心（centroid）。

演算法很簡單：隨機初始化群心 → 分配資料到最近的群心 → 重新計算群心 → 重複。但 K 值的選擇是個挑戰，常用肘部法（Elbow Method）。

降維

  • PCA：找到方差最大的方向投影，線性降維
  • t-SNE：非線性降維，擅長視覺化高維資料（但不適合做特徵轉換）
  • UMAP：更快的非線性降維替代方案

如何選擇模型？

場景推薦模型
結構化資料、追求效能XGBoost / LightGBM
需要可解釋性線性模型 / 決策樹
高維稀疏資料邏輯回歸 / SVM
探索性分析隨機森林（特徵重要性）
小資料集SVM / KNN

實務心法

  1. 永遠先試簡單模型：線性模型作為基準線，再逐步複雜化
  2. 結構化資料優先考慮 Gradient Boosting，不要一上來就深度學習
  3. 可解釋性有時比效能更重要，尤其在醫療、金融等領域
  4. 特徵工程往往比模型選擇更重要——好的特徵 + 簡單模型 > 差的特徵 + 複雜模型
