模型太複雜會過擬合，太簡單會欠擬合。正則化和特徵選擇，就是在這兩個極端之間找到平衡的藝術。

過擬合：機器學習的頭號公敵

過擬合（Overfitting）指的是模型在訓練資料上表現很好，但在新資料上表現很差。模型把訓練資料中的噪聲也學進去了，失去了泛化能力。

如何偵測過擬合？

最直接的方法：比較訓練誤差和驗證誤差。如果訓練誤差持續下降但驗證誤差開始上升，就是過擬合的信號。

過擬合的根本原因

  1. 模型太複雜：參數太多，遠超資料量
  2. 資料太少：不足以支撐模型的複雜度
  3. 訓練太久：模型開始記憶噪聲

正則化：給模型加上「約束」

正則化的核心思想：在損失函數中加入一個懲罰項，限制模型參數的大小，從而降低模型複雜度。

L1 正則化（Lasso）

懲罰參數的絕對值之和。

L1 正則化的獨特之處在於它會產生稀疏解——把一些參數直接壓縮到零。這等於自動做了特徵選擇，留下最重要的特徵。

直覺理解：L1 在參數空間中劃出一個菱形約束區域。最優解往往落在菱形的角上，而角上必定有些座標為零。

L2 正則化（Ridge）

懲罰參數的平方和。

L2 不會把參數壓成零，而是讓所有參數都變小。它的效果是讓模型更「平滑」，減少對單一特徵的依賴。

直覺理解：L2 在參數空間中劃出一個圓形約束區域。最優解不太可能落在座標軸上，所以參數不會為零。

Elastic Net

L1 和 L2 的結合。同時具備稀疏性和穩定性。當特徵之間有高度相關性時，Elastic Net 比純 L1 更穩健。

Dropout

深度學習專用的正則化技巧：在訓練時隨機「關閉」一定比例的神經元。

為什麼有效？每次訓練都等於在用一個不同的子網路。最終效果類似於訓練了很多個模型再做平均（類似集成學習）。

其他正則化技巧

技巧機制適用場景
Early Stopping驗證誤差不再下降時停止訓練所有模型
資料增強人為擴大訓練資料影像、文字
Batch Normalization穩定每層的輸入分佈深度學習
Label Smoothing把硬標籤變成軟標籤分類任務
Weight Decay直接在參數更新時衰減權重深度學習

特徵選擇：找出真正重要的特徵

並非所有特徵都有用。無關特徵會增加模型複雜度、拖慢訓練速度，甚至降低效能（維度災難）。

三大策略

過濾法（Filter）： 在訓練模型之前，根據統計指標篩選特徵。

  • 相關係數：與目標變數的線性相關程度
  • 卡方檢定：類別特徵與目標的關聯性
  • 互資訊（Mutual Information）：特徵與目標的非線性關聯

包裝法（Wrapper）： 把模型效能作為評估標準，反覆嘗試不同特徵組合。

  • 前向選擇：從零開始，逐步加入最有用的特徵
  • 後向消除：從全部特徵開始，逐步移除最無用的
  • 計算成本高，但通常效果最好

嵌入法（Embedded）： 模型訓練過程中自動選擇特徵。

  • L1 正則化自動產生稀疏解
  • 決策樹的特徵重要性
  • XGBoost 的 feature importance

特徵重要性的評估

  • Permutation Importance：打亂某個特徵的值，看模型效能下降多少。下降越多，該特徵越重要。
  • SHAP Values：基於賽局理論，公平地分配每個特徵對預測的貢獻。是目前最被推薦的方法。

偏差-方差權衡

正則化和特徵選擇的核心，都在處理偏差-方差權衡（Bias-Variance Tradeoff）：

  • 高偏差：模型太簡單，學不到資料的模式（欠擬合）
  • 高方差：模型太複雜，對訓練資料過度敏感（過擬合）
  • 目標：找到偏差和方差的最佳平衡點

實務心法

  1. 正則化幾乎永遠是好主意——它是防止過擬合成本最低的方法
  2. 先用 L2 正則化，如果你需要特徵選擇再考慮 L1 或 Elastic Net
  3. 特徵選擇在特徵很多（>100）時特別重要，能顯著提升模型效能和訓練速度
  4. 交叉驗證是判斷正則化強度的最佳工具——讓資料告訴你答案
