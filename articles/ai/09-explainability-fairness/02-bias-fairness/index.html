你的模型可能在不知不覺中歧視某些群體。AI 偏差不只是技術問題，更是社會正義問題。識別和緩解偏差，是負責任 AI 的核心。

偏差的來源

資料偏差

歷史偏差：歷史資料反映了過去的歧視。例如，過去女性較少擔任高階職位的資料，會讓模型學到「女性不適合升遷」。

代表性偏差：訓練資料沒有均勻覆蓋所有群體。例如，醫療 AI 主要用白人患者的資料訓練，對其他種族的表現就較差。

標注偏差：標注者的主觀判斷可能帶有偏見。

測量偏差：不同群體的特徵測量方式或品質不同。

演算法偏差

優化目標偏差：模型優化整體準確率，但忽略少數群體的表現。

特徵選擇偏差：看似中性的特徵可能是敏感屬性的代理變數。例如，郵遞區號可能高度相關於種族。

公平性的定義

公平性沒有唯一的定義。不同的定義之間甚至可能互相矛盾。

統計公平性

人口統計學均等（Demographic Parity）：不同群體獲得正面預測的比例相同。

問題：不考慮真實能力的差異。

機會均等（Equal Opportunity）：在真正的正類中，不同群體被正確預測為正的比例相同（即 True Positive Rate 相同）。

等化勝算（Equalized Odds）：不同群體的 True Positive Rate 和 False Positive Rate 都相同。

個體公平性

相似的個體應該獲得相似的預測結果。問題是：如何定義「相似」？

因果公平性

改變敏感屬性（如性別）不應該改變預測結果。但需要因果模型來分析。

不可能定理

數學上已經證明：大多數公平性定義不能同時滿足（除非模型完美或群體完全相同）。你必須根據場景選擇最重要的公平性定義。

偏差偵測

分群分析

把模型效能按敏感屬性（性別、年齡、種族等）分群報告。看不同群體之間的效能差距。

公平性指標

指標定義合理範圍
Disparate Impact不利群體正面率 / 有利群體正面率> 0.8（四五分之四規則）
Equal Opportunity DifferenceTPR 差距越接近 0 越好
Average Odds DifferenceTPR 和 FPR 差距的平均越接近 0 越好

工具

  • Fairlearn（微軟）：公平性評估和緩解的 Python 庫
  • AI Fairness 360（IBM）：全面的偏差偵測和緩解工具包
  • What-If Tool（Google）：互動式模型分析工具

偏差緩解策略

前處理（Pre-processing）

在訓練之前修正資料：

  • 重新加權：給不同群體的樣本不同的權重
  • 重新採樣：平衡各群體的樣本數量
  • 資料增強：為少數群體生成更多樣本

訓練中（In-processing）

在訓練過程中加入公平性約束：

  • 對抗式去偏：訓練模型同時不讓它預測敏感屬性
  • 公平性約束優化：在損失函數中加入公平性懲罰項
  • 多目標優化：同時優化效能和公平性

後處理（Post-processing）

在模型輸出上做調整：

  • 閾值調整：對不同群體使用不同的決策閾值
  • 校準：確保不同群體的預測機率具有相同含義

公平性 vs 效能

公平性約束通常會犧牲一些整體效能。這是一個價值判斷，不是技術判斷。

關鍵問題：「我們願意為了公平性，犧牲多少效能？」

實務心法

  1. 公平性不是事後補救——從資料收集階段就開始考慮
  2. 選擇適合你場景的公平性定義——不要試圖同時滿足所有定義
  3. 分群分析是最基本的檢查——每個生產模型都應該做
  4. 公平性是持續的工作——需要定期監控，而非一次性檢查
