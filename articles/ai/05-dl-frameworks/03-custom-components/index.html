當預設的元件無法滿足你的需求時，你需要自己動手。自定義元件開發是從「使用框架」到「精通框架」的關鍵跨越。

為什麼需要自定義元件？

標準元件覆蓋了大部分常見需求，但在以下場景你會需要自定義：

  • 研究新的網路架構或損失函數
  • 實作論文中尚未被框架收錄的方法
  • 針對特定業務邏輯的特殊需求
  • 效能優化（融合多個操作為一個自定義核心）

自定義層的設計原則

封裝性

一個好的自定義層應該像一個黑箱：

  • 清楚定義輸入和輸出的形狀
  • 內部狀態（參數）由層自己管理
  • 與框架的其他元件無縫整合

參數管理

可訓練的參數需要正確註冊，這樣框架才能：

  • 在反向傳播時計算梯度
  • 在儲存模型時包含這些參數
  • 在分散式訓練時同步這些參數

初始化策略

好的參數初始化能避免梯度消失或爆炸：

初始化方法適用場景
Xavier/GlorotSigmoid、Tanh 啟動函數
Kaiming/HeReLU 系列啟動函數
正交初始化RNN 的循環權重
常數初始化偏差項（通常初始化為 0）

自定義損失函數

設計要點

  1. 可微性：損失函數必須對模型輸出可微（至少幾乎處處可微）
  2. 數值穩定性：避免 log(0) 或除以零。加入小的 epsilon。
  3. 批次處理：支援整個批次的計算，而非逐樣本
  4. 梯度合理性：梯度不應該消失或爆炸

常見自定義模式

加權損失：不同類別或不同樣本使用不同的權重。

組合損失：多個損失函數的加權和。例如分割任務常用交叉熵 + Dice Loss。

課程損失：隨著訓練進度改變損失函數的行為。例如 Focal Loss 的 gamma 可以動態調整。

自定義訓練迴圈

框架的預設訓練迴圈（如 Keras 的 .fit()）方便但不夠彈性。自定義訓練迴圈讓你完全控制：

典型結構

  1. 遍歷 DataLoader
  2. 前向傳播
  3. 計算損失
  4. 反向傳播
  5. 梯度裁剪（可選）
  6. 參數更新
  7. 記錄指標
  8. 驗證評估

進階技巧

混合精度訓練：用 float16 加速計算，同時用 float32 保持數值穩定。

梯度累積：當 GPU 記憶體不夠時，分多個小批次累積梯度，模擬大批次效果。

梯度裁剪：限制梯度的最大值，防止梯度爆炸。

模型指數移動平均（EMA）：維護模型參數的移動平均版本，通常比最終模型效能更穩定。

除錯技巧

形狀問題

最常見的錯誤。建議：

  • 在每個自定義元件中加入形狀斷言
  • 用小批次測試每一步的輸入輸出形狀

梯度問題

  • 用框架的梯度檢查工具驗證自定義元件的梯度
  • 檢查是否有計算圖斷開的情況（detach、no_grad）

數值問題

  • 使用 torch.isnan() 和 torch.isinf() 檢查 NaN 和 Inf
  • 在除法和 log 中加入 epsilon

實務心法

  1. 先用標準元件，確認標準元件不滿足需求後再自定義
  2. 自定義元件一定要寫單元測試——特別是梯度正確性
  3. 效能不好時先檢查是否有不必要的資料拷貝或 CPU-GPU 同步
  4. 參考知名開源專案的實作——比自己摸索快很多
