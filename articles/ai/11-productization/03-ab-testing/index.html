離線指標再好，都不能保證上線效果。A/B 測試是驗證模型在真實環境中效果的金標準。

A/B 測試的基本原理

把用戶隨機分成兩組（或多組）：

  • 對照組（Control）：使用現有模型或方案
  • 實驗組（Treatment）：使用新模型或方案

比較兩組的業務指標差異。如果差異足夠大且統計顯著，就可以確信新方案更好。

為什麼離線指標不夠？

離線評估和線上行為之間可能有巨大的差距：

  • 離線資料是歷史的，不反映未來
  • 離線無法捕捉用戶與模型的互動效應
  • 離線指標（如 AUC）與業務指標（如營收）不一定正相關

A/B 測試的設計

確定假設

在開始實驗之前，明確：

  • 假設：新模型會提高指標 X
  • 主要指標：你最在意的業務指標
  • 護欄指標：不能惡化的指標（如用戶投訴率）
  • 方向：期望的影響方向和大小

樣本量計算

在實驗開始前，計算需要多少用戶和多長時間才能偵測到有意義的差異。

影響因素：

  • 效果大小（Effect Size）：期望的改善幅度。越小，需要越多樣本。
  • 顯著性水準（α）：通常 0.05（5% 的假陽性風險）
  • 統計功效（Power）：通常 0.8（80% 的機率偵測到真實效果）
  • 指標的方差：方差越大，需要越多樣本

隨機分組

好的隨機分組應該：

  • 每個用戶穩定在同一組（不會看到兩個版本）
  • 各組的用戶特徵分佈均勻
  • 不受時間和地域因素影響

通常用用戶 ID 的雜湊值來決定分組。

實驗分析

統計顯著性

用假設檢定判斷結果是否可信：

  • 計算 p 值：觀察到的差異在虛無假設（兩組無差異）下出現的機率
  • p < 0.05 通常被認為是顯著的
  • 但要注意：統計顯著 ≠ 實際重要

信賴區間

比 p 值更有資訊量。告訴你效果的可能範圍。

例如：「點擊率提升 2.3%，95% 信賴區間 [1.1%, 3.5%]」

常見陷阱

早期偷看（Peeking）：反覆檢查結果，看到顯著就停止。這會嚴重膨脹假陽性率。

辛普森悖論：整體看有效，但分群看可能無效（或反過來）。

假陽性累積：同時測試很多指標，總會有一些碰巧顯著。用 Bonferroni 校正。

選擇偏差：分組不夠隨機，導致兩組先天就有差異。

進階方法

多臂老虎機（Multi-Armed Bandit）

在實驗過程中動態調整各組的流量分配：效果好的方案自動獲得更多流量。

好處：減少實驗期間的「損失」（不用把 50% 流量給可能較差的方案）。 代價：統計分析更複雜，可能需要更長時間才能確定結果。

交錯實驗（Interleaving）

適合排序/推薦系統。在同一個用戶的結果中混合兩個模型的推薦，看用戶更傾向點擊哪些。

比傳統 A/B 測試更靈敏，需要更少的用戶量。

漸進式發布

1% → 5% → 25% → 50% → 100%

在每個階段確認效果和穩定性，再擴大範圍。

實務心法

  1. 每個上線的模型變更都應該經過 A/B 測試——不要靠直覺
  2. 在實驗開始前就確定分析計畫——不要事後挑好看的指標
  3. 給實驗足夠的時間——不要因為急著上線就提前結束
  4. 記錄所有實驗結果——失敗的實驗也有價值
