真實世界的資料永遠是髒的。缺失值、異常值、格式不一致、重複記錄⋯⋯資料清洗可能佔據資料科學家 60-80% 的時間。但它不是苦差事，而是決定模型成敗的關鍵步驟。

常見的資料品質問題

問題類型範例影響
缺失值欄位為 NULL 或空白模型無法處理，或產生偏差
重複記錄同一筆訂單出現兩次統計結果膨脹
格式不一致日期格式混用、大小寫不一無法正確合併或分組
異常值年齡為 -5 或 999扭曲統計量和模型訓練
類型錯誤數字存為字串計算失敗

缺失值處理策略

處理缺失值沒有萬靈丹，選擇取決於缺失的原因和比例。

缺失機制

理解「為什麼資料會缺失」很重要：

  • 完全隨機缺失（MCAR）：缺失與任何變數無關。可以安全地刪除。
  • 隨機缺失（MAR）：缺失與其他觀察到的變數有關。可以用這些變數來插補。
  • 非隨機缺失（MNAR）：缺失本身就包含資訊。例如高收入者不願意填寫收入。

處理方法

刪除策略：

  • 刪除含缺失值的行：最簡單，但如果缺失比例高，會損失大量資料
  • 刪除缺失比例高的欄位：如果某欄位 80% 都是缺失，可能沒有保留價值

插補策略：

  • 均值/中位數/眾數填補：簡單但會低估資料的變異性
  • 前向/後向填補：適合時間序列資料
  • KNN 插補：用最相似的樣本來推估缺失值
  • 多重插補（MICE）：更精確但計算成本高

保留策略：

  • 新增「是否缺失」的指標欄位：讓模型自己學習缺失的意義
  • 某些模型（如 XGBoost）天生能處理缺失值

異常值偵測與處理

偵測方法

  • 統計法：超過均值 ± 3 個標準差的值。前提是資料近似常態分佈。
  • IQR 法：Q1 - 1.5×IQR 到 Q3 + 1.5×IQR 之外的值。更穩健。
  • Isolation Forest：ML 方法，適合高維資料。
  • 視覺化：箱型圖、散布圖，最直覺的方法。

處理方法

  • 刪除：確定是資料錯誤時
  • 截斷（Clipping）：把極端值限制在某個範圍內
  • 轉換：取對數等方式壓縮極端值的影響
  • 分別建模：如果異常值有特殊意義，可以分開處理

資料轉換

數值特徵轉換

  • 標準化（Standardization）：轉換為均值 0、標準差 1。適合假設常態分佈的演算法。
  • 正規化（Normalization）：縮放到 [0, 1] 區間。適合距離型演算法（KNN、SVM）。
  • 對數轉換：處理右偏分佈，讓分佈更接近常態。
  • Box-Cox 轉換：自動找到最佳的冪次轉換。

類別特徵轉換

  • One-Hot Encoding：每個類別變一個二元欄位。類別少時適用。
  • Label Encoding：類別轉數字。只適合有序類別（如低/中/高）。
  • Target Encoding：用目標變數的統計量替代類別。效果好但容易過擬合，需要搭配交叉驗證。
  • Frequency Encoding：用類別的出現頻率替代。

文字資料轉換

  • 分詞（Tokenization）：把文字切成詞語或子詞
  • 去除停用詞：刪掉「的」、「了」等無意義的詞
  • TF-IDF：衡量詞語在文件中的重要程度
  • Embedding：用預訓練模型把文字轉成向量

資料清洗的最佳實踐

  1. 先探索再清洗：用統計摘要和視覺化理解資料，再動手
  2. 保留原始資料：清洗結果存為新版本，不覆蓋原始資料
  3. 記錄每一步操作：可重現的清洗流程，用程式碼而非手動操作
  4. 設定資料品質檢查點：清洗後驗證資料是否符合預期
  5. 與領域專家合作：有些「異常值」可能是真實的極端情況
