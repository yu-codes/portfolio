概述

  • ECR（Elastic Container Registry）：AWS 管理的容器映像倉庫，類似 Docker Hub，可推送與拉取 Docker 映像
  • ECS（Elastic Container Service）：容器編排服務，可建立、執行、管理 Docker 容器叢集

ECS 核心概念

Task Definition（任務定義）

定義單一或多個容器的設定內容：

  • 映像來源
  • 記憶體和 CPU
  • 埠口映射
  • 環境變數
  • 儲存配置

IAM Role 類型

類型說明
ECS Task IAM Role給容器使用，讓其可存取 AWS 資源（如 S3、DynamoDB）
Task Execution IAM Role給 EC2 執行階段使用，用於拉映像、發送 log 至 CloudWatch

Service Definition（服務定義）

  • 用來部署與維護指定數量的 ECS 任務
  • 可搭配 CLB/ALB
  • 執行在 EC2 叢集或 Fargate 上

啟動類型

EC2 啟動類型

特性說明
管理方式自行管理 EC2 實例
Agent需安裝 ECS Agent
適用場景長時間、可預測工作量
成本較低
控制權完全控制底層基礎設施

ECS Agent

  • 安裝於 EC2 實例中
  • 負責回報資源與任務狀態
  • 接收 ECS 控制指令

Fargate 啟動類型

特性說明
管理方式Serverless，ECS 管理底層 EC2
配置僅需定義容器資源（CPU/記憶體）
適用場景間歇或彈性工作
成本較高
控制權無需管理基礎設施

EC2 vs Fargate 比較

項目EC2Fargate
基礎設施管理自行管理AWS 管理
擴展需管理 EC2 擴展自動擴展
成本較低（可用 Reserved/Spot）較高
啟動時間較慢較快
適用場景穩定工作負載彈性工作負載

EKS（Elastic Kubernetes Service）

  • AWS 託管的 Kubernetes（K8s）叢集服務
  • 適合已有 K8s 經驗或使用原生 K8s 的團隊

ECS vs EKS

項目ECSEKS
編排引擎AWS 原生Kubernetes
學習曲線較低較高
跨雲僅 AWS支援多雲
生態系統AWS 生態K8s 生態
成本較低較高（控制平面費用）

ECS 網路模式

模式說明
bridge預設模式，使用 Docker bridge 網路
host直接使用主機網路
awsvpc每個任務有獨立 ENI（推薦）
none無網路連接

服務發現

  • 與 Route 53 整合
  • 自動註冊和取消註冊服務
  • 支援 DNS 和 SRV 記錄

Auto Scaling

Service Auto Scaling

  • Target Tracking：維持指標在目標值
  • Step Scaling：階梯式調整
  • Scheduled Scaling：排程調整

常用指標

  • ECSServiceAverageCPUUtilization
  • ECSServiceAverageMemoryUtilization
  • ALBRequestCountPerTarget

ECR（Elastic Container Registry）進階功能

映像管理

  • 映像標籤：支援可變和不可變標籤
  • 跨區域/跨帳號複製：自動同步映像到其他區域或帳號
  • 映像掃描：自動掃描容器映像的安全漏洞
  • 生命週期政策：自動清理舊映像

安全功能

  • 支援 KMS 加密
  • IAM 細粒度存取控制
  • VPC 端點支援私有存取

EKS（Elastic Kubernetes Service）進階功能

Kubernetes 編排

  • AWS 託管的控制平面，自動升級和修補
  • 支援多個 Kubernetes 版本
  • 與 AWS 服務原生整合（IAM、VPC、ALB/NLB）

安全性

  • IAM Roles for Service Accounts (IRSA)：為 Pod 分配 IAM 角色
  • Pod 安全群組：直接對 Pod 套用安全群組
  • Secrets 加密：使用 KMS 加密 etcd 中的 Secrets

負載平衡

  • AWS Load Balancer Controller 支援 ALB/NLB
  • 支援 Ingress 資源自動配置 ALB

監控

  • Container Insights 提供叢集和 Pod 指標
  • 與 CloudWatch 和 X-Ray 整合

EKS Anywhere

  • 在本地資料中心運行 EKS
  • 使用相同的工具和 API
  • 支援 VMware vSphere

ECS Anywhere

  • 在本地伺服器或虛擬機上運行 ECS 任務
  • 使用 ECS Agent 連接到 AWS
  • 統一管理雲端和本地容器工作負載

最佳實踐

  1. 使用 Fargate 簡化管理：無需管理基礎設施
  2. 使用 awsvpc 網路模式：更好的網路隔離
  3. 配置適當的 Task 資源：避免資源浪費
  4. 實施 Service Auto Scaling：自動調整任務數量
  5. 使用 ECR：與 ECS 無縫整合
  6. 分離 Task Role 和 Execution Role：遵循最小權限原則
  7. 啟用 ECR 映像掃描：主動發現安全漏洞
  8. 使用 IRSA（EKS）：為 Pod 配置最小權限

容器基礎概念

虛擬化回顧

  • EC2 主機運行多個隔離的虛擬機
  • Hypervisor 隔離各 VM
  • 每個 VM 有自己的 OS
  • 作業系統佔用空間（記憶體、CPU、磁碟）
  • 雖然不會 100% 複製 OS 使用量，但會消耗一些資源

容器化

  • 與虛擬機不同：容器使用主機作業系統
  • 容器不如虛擬機隔離
  • 容器僅包含應用程式和相依套件
  • 容器引擎（Docker）讓容器相信它們有自己的作業系統

容器 vs 虛擬機

項目容器虛擬機
啟動時間秒級分鐘級
資源使用輕量較重
隔離性程序級完全隔離
OS共用主機 OS獨立 OS
可移植性極高較低

映像與容器

  • Docker 映像：用於建立容器的藍圖
  • 透過 Dockerfile 建立
  • 映像分為多層
  • 只有頂層（讀寫層）可修改
  • 映像儲存在容器登錄庫（如 Docker Hub、ECR）

ECS 概念詳解

架構

  • 容器定義：定義映像和埠口
  • 任務定義：容器組成的應用，類似 Pod
  • 任務角色：任務獲得的 IAM 角色
  • 服務：長時間運行的應用，如 Web 服務

叢集模式

模式說明
EC2 模式ECS 管理 EC2 實例，您管理容量
Fargate 模式無需管理伺服器

EC2 vs ECS（EC2）vs Fargate

適用場景

選項適用場景
EC2需要長期運行的計算
ECS（EC2）需要 Docker 容器，需控制底層 EC2
Fargate需要 Docker 容器但不想管理伺服器

管理與開銷比較

項目EC2ECS（EC2）Fargate
基礎設施管理完全自管部分自管AWS 管理
容量規劃需要需要自動
OS 修補需要需要不需要
擴展複雜度高中低

ECR（Elastic Container Registry）詳解

功能

  • AWS 提供的託管容器映像登錄庫
  • 每個區域每個帳號都有一個登錄庫
  • 內有多個儲存庫
  • 每個儲存庫可以有多個映像
  • 映像可以有多個標籤

與 ECS 整合

  • ECS 原生支援 ECR
  • 使用 Task Execution Role 拉取映像
  • 支援 VPC 端點私有存取

Kubernetes 101

概述

  • 開源容器編排系統
  • 雲端無關
  • 提供一種方式讓多個容器一起運行
  • 自動化部署、擴展和管理

核心架構

Cluster → Control Plane + Nodes

控制平面組件

組件說明
kube-apiserver叢集的 API 入口
etcd鍵值存儲，保存叢集狀態
kube-scheduler決定 Pod 放置位置
kube-controller-manager管理控制器
cloud-controller-manager雲端提供商整合

節點組件

組件說明
kubelet在每個節點上運行，管理 Pod
kube-proxy網路代理和負載平衡
Container Runtime運行容器（如 containerd）

重要概念

概念說明
Pod最小部署單位，包含一或多個容器
Deployment管理 Pod 的期望狀態
Service暴露 Pod 的網路抽象
ReplicaSet維護 Pod 副本數量
IngressHTTP(S) 路由規則

EKS 架構詳解

概述

  • AWS 託管 Kubernetes
  • 可運行在 AWS、Outposts、EKS Anywhere、Local Zones
  • 控制平面運行在 AWS 託管的 VPC 中
  • 整合 AWS 服務：ECR、ELB、IAM、VPC

控制平面

  • AWS 管理多個 AZ
  • etcd 跨多個 AZ 分散
  • 透過 ENI 注入客戶 VPC

EKS 節點類型

類型說明
自管節點您建立和管理的 EC2
託管節點群組AWS 管理的 EC2 群組
Fargate Pods無伺服器容器

節點配置

  • 需確認 EKS 已優化的 AMI
  • 或使用 EKS Distro
  • 透過 ConfigMap 連接到 EKS

ECS vs EKS 選擇指南

考量因素選 ECS選 EKS
學習曲線較陡Kubernetes 經驗
雲端鎖定接受 AWS 鎖定需要可移植性
生態系統AWS 原生Kubernetes 生態
複雜度較簡單較複雜
成本較低控制平面費用

容器安全最佳實踐

映像安全

  1. 使用官方基底映像
  2. 定期掃描安全漏洞
  3. 使用不可變標籤
  4. 最小化映像大小

運行時安全

  1. 使用非 root 使用者
  2. 限制容器權限
  3. 使用唯讀檔案系統
  4. 配置資源限制

網路安全

  1. 使用 awsvpc 網路模式
  2. 配置安全群組
  3. 使用私有子網路
  4. 啟用 VPC Flow Logs
