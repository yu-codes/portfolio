ASG（Auto Scaling Group）概述

  • 根據 CPU、網路或自訂指標或排程，自動調整 EC2 數量
  • 設定最小 / 最大 / 期望容量，確保應用彈性擴縮容
  • 使用 Launch Template（新版）或 Launch Configuration（舊版）建立實例

Launch Template vs Launch Configuration

項目Launch TemplateLaunch Configuration
版本支援支援版本管理不支援
修改可修改無法修改，只能重建
實例組合支援混合實例不支援
Spot/On-Demand支援組合不支援

Scaling Policy（擴縮策略）

策略類型說明範例
Target Tracking維持指標在目標值平均 CPU 利用率在 40%
Simple Scaling單一條件觸發CPU >80% 加 2 台
Step Scaling依條件階梯調整CPU 60-80% 加 1 台，>80% 加 3 台
Scheduled Action排定時間調整每週五 5pm 加到 10 台

Target Tracking 常用指標

  • ASGAverageCPUUtilization
  • ASGAverageNetworkIn
  • ASGAverageNetworkOut
  • ALBRequestCountPerTarget

Termination Policy（終止策略）

預設流程

  1. 找出 EC2 最多的 AZ
  2. 刪除該 AZ 中使用最舊啟動模板的實例
  3. 若並列，刪除最接近下個計費時段的

可選策略

策略說明
Default預設策略
OldestInstance最舊的實例
NewestInstance最新的實例
OldestLaunchConfiguration最舊啟動配置
OldestLaunchTemplate最舊啟動模板
ClosestToNextInstanceHour最接近下個計費時段
AllocationStrategy根據配置策略

Cooldown / Grace Period

設定說明
Cooldown調整後冷卻期，避免過度擴縮
Health Check Grace Period進入服務後的健康檢查緩衝期

Lifecycle Hooks

可在擴增或縮減時加入自定邏輯：

擴增時（Launching）

  • 安裝軟體
  • 配置應用程式
  • 註冊到服務

縮減時（Terminating）

  • 備份資料
  • 上傳 log
  • 取消註冊

狀態

狀態說明
Pending:Wait等待 Hook 完成
Pending:Proceed繼續啟動
Terminating:Wait等待 Hook 完成
Terminating:Proceed繼續終止

健康檢查類型

類型說明
EC2EC2 實例狀態檢查
ELBELB 健康檢查
Custom自訂健康檢查

與 ELB 整合

  • ASG 可自動將新實例註冊到 Target Group
  • 不健康實例會被終止並替換
  • 支援多個 Target Group

擴縮最佳實踐

  1. 使用 Target Tracking：最簡單且有效的擴縮策略
  2. 設定適當的 Cooldown：避免不必要的擴縮
  3. 配置 Health Check Grace Period：給應用足夠啟動時間
  4. 使用 Lifecycle Hooks：確保安全的擴縮流程
  5. 跨多 AZ 部署：提高可用性
  6. 使用 Launch Template：更靈活的實例配置

Scaling Policy 詳解

Simple Scaling

  • 一個條件 → 一個動作
  • 有 Cooldown 期間
  • 不建議：使用 Step Scaling 代替

Step Scaling

範圍動作
40-60% CPU+1 實例
60-80% CPU+2 實例
>80% CPU+4 實例
  • 多個步驟，更精細控制
  • 建議用於詳細控制

Target Tracking

  • 指定目標值（如 CPU 40%）
  • ASG 自動調整以維持目標
  • 最簡單的方式

Scheduled Scaling

使用案例說明
營業時間9am 擴展，6pm 縮減
週末減少容量
促銷提前擴展

Lifecycle Hooks 詳解

架構

Pending → Pending:Wait → Pending:Proceed → InService
                ↓
         自訂動作（SNS/SQS/Lambda）

擴展 Hook 使用案例

動作說明
安裝軟體安裝應用程式
提取配置從 S3 或 Parameter Store
註冊服務向服務發現註冊
準備資料預熱快取

終止 Hook 使用案例

動作說明
備份上傳日誌到 S3
取消註冊從服務發現移除
通知發送通知
清理刪除暫存資料

整合

  • EventBridge：觸發 Lambda
  • SNS：發送通知
  • SQS：佇列處理

健康檢查詳解

檢查類型

類型說明
EC2EC2 狀態檢查
ELBELB 健康檢查
Custom自訂（透過 API）

ELB 健康檢查

  • 比 EC2 狀態檢查更應用層級
  • 實例可能 EC2 健康但應用不健康
  • 建議：ELB + EC2 結合

Health Check Grace Period

  • 新實例的寬限期
  • 避免啟動期間被標記為不健康
  • 預設：300 秒

ASG + ELB 整合

架構

流量 → ELB → Target Group ← ASG 自動註冊/取消註冊

自動註冊

  • 新實例自動註冊到 Target Group
  • 需指定 Target Group ARN

健康檢查整合

  • ASG 使用 ELB 健康檢查
  • 不健康實例被終止並替換

進階功能

Instance Refresh

  • 滾動更新實例
  • 指定更新的百分比
  • 自動替換舊實例

Warm Pools

  • 預熱實例池
  • 減少擴展時間
  • 實例處於停止或運行狀態

Predictive Scaling

  • 使用 ML 預測需求
  • 提前擴展
  • 適合可預測的流量模式
