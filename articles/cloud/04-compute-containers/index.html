Compute services are where your code runs. From virtual machines to containers to serverless functions, understanding compute options helps you choose the right tool for each workload.

Why Compute Knowledge Matters

The right compute choice affects:

  • Cost efficiency
  • Operational complexity
  • Scaling behavior
  • Development velocity

Subdomains

EC2 & Virtual Machines

Select instance types, understand pricing models, implement auto-scaling groups, and optimize for cost/performance.

Container Orchestration (EKS/GKE/AKS)

Deploy to managed Kubernetes, understand control planes vs worker nodes, and implement production-ready clusters.

Container Basics

Build and run Docker containers, understand image registries, implement container security scanning.

Serverless Computing

Design Lambda functions, understand cold starts, implement event-driven architectures, and optimize performance.

Spot/Preemptible Instances

Reduce costs with spot instances, handle interruptions gracefully, and design fault-tolerant workloads.

Batch & HPC

Run batch workloads with AWS Batch or Cloud Run Jobs, implement parallel processing at scale.
