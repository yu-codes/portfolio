概述

CI/CD 是現代軟體開發的核心實踐，AWS 提供多種服務支援自動化部署流程。

AWS Step Functions（狀態機）

  • 無伺服器的流程編排服務
  • 使用 JSON 宣告式語法定義狀態機，可連接多個 Lambda 或其他服務流程
  • 適合用於編排複雜的部署流程

AWS Simple Workflow Service（SWF）

  • 執行於 EC2 的流程編排服務（非 Serverless）
  • 支援流程中的外部訊號、子流程回傳值等進階需求
  • 新案建議改用 Step Functions

部署策略

CloudFormation 部署

  • 使用模板定義完整基礎設施
  • 支援 Stack 更新與回滾
  • StackSets 支援跨帳號、跨區域部署

Elastic Beanstalk 部署

  • 支援多種部署策略：
    • All at once：一次全部更新
    • Rolling：滾動更新
    • Rolling with additional batch：額外批次滾動
    • Immutable：不可變部署
    • Blue/Green：藍綠部署

與其他 AWS 服務整合

自動化觸發

  • EventBridge：可排程觸發部署任務（類似 cron）
  • Lambda：可作為部署流程中的自動化腳本執行
  • SNS/SQS：部署通知與訊息佇列

監控與回滾

  • CloudWatch：監控部署過程中的指標
  • CloudTrail：稽核部署操作歷史
  • Auto Scaling：部署後自動調整容量

AWS CodePipeline

  • 完全託管的持續交付服務
  • 自動化發布和部署流程
  • 支援來源、建置、測試、部署多個階段

階段類型

階段說明
Source從 CodeCommit、GitHub、S3 取得原始碼
Build使用 CodeBuild 編譯和測試
Test執行額外的測試
Deploy使用 CodeDeploy、Beanstalk、CloudFormation 部署

功能

  • 支援手動審批階段
  • 與 SNS 整合通知
  • 透過 CloudTrail 稽核

AWS CodeBuild

  • 完全託管的持續整合服務
  • 編譯、測試和打包程式碼
  • 使用 buildspec.yml 定義建置步驟

特性

  • 支援多種程式語言和框架
  • 支援自訂 Docker 映像
  • 建置快取加速後續建置
  • 支援 VPC 存取私有資源

AWS CodeDeploy

  • 自動化應用程式部署服務
  • 支援 EC2、Lambda、ECS 和本地伺服器

部署策略

策略說明適用目標
In-place原地更新EC2、本地伺服器
Blue/Green藍綠部署EC2、ECS
Linear線性漸進部署Lambda
Canary金絲雀部署Lambda

功能

  • 自動回滾失敗的部署
  • 使用 AppSpec 檔案定義部署
  • 與 ELB 整合流量切換

AWS CodeCommit

  • 完全託管的 Git 儲存庫服務
  • 安全、高可用、可擴展
  • 與 IAM 整合細粒度存取控制

功能

  • 支援所有標準 Git 功能
  • Pull Request 和程式碼審查
  • 觸發器連接 Lambda 或 SNS
  • 跨區域複製

CI/CD 最佳實踐

  1. 基礎設施即程式碼：所有資源定義存放於版本控制
  2. 自動化測試：部署前執行自動化測試
  3. 漸進式部署：使用 Rolling 或 Blue/Green 策略降低風險
  4. 監控與告警：部署後持續監控應用健康狀態
  5. 快速回滾機制：確保能快速還原至穩定版本
  6. 使用 CodePipeline：統一管理完整 CI/CD 流程
  7. 啟用手動審批：生產環境部署前審核
