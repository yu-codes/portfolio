AWS CloudFormation

  • 基礎建設即程式碼（Infrastructure as Code，IaC）
  • 使用 YAML 或 JSON 格式定義模板（Template），可建立、更新、刪除整組 AWS 資源（Stack）
  • 組成包括：Resources（必填）、Parameters、Mappings、Outputs、Conditions、Metadata
  • 可設定 DependsOn 控制資源建立順序
  • 可設定 DeletionPolicy 為 retain（保留資源）、snapshot（備份）
  • 支援 EC2 啟動時執行 bootstrap script 安裝套件
  • 預設啟用「錯誤自動回滾」，若 Stack 建立失敗會還原
  • CloudFormation StackSets 可讓你一次跨帳號、跨區域建立多組 Stack
  • 本身免費，僅對應 AWS 資源計費
  • 用途：建立多環境（開發 / 測試 / 生產）、每日建立刪除資源以節省費用

AWS Elastic Beanstalk

  • 平台即服務（PaaS），簡化應用程式部署與管理
  • 自動處理容量配置、負載平衡、自動擴展與應用健康監控
  • 支援多種平台：Java (Tomcat)、PHP、Python、Node.js、Ruby、.NET、Docker
  • 支援 Web Tier（HTTP 服務）與 Worker Tier（處理背景任務）
  • 本身免費，僅對應資源如 EC2、ELB、ASG 計費
  • 支援自訂 AMI、支援多版本佈署與版本回滾

AWS OpsWorks

  • 託管 Chef 與 Puppet 的設定管理服務（Configuration as Code）
  • 可用來自動佈署、配置與維護 EC2 環境
  • 支援 Stack 架構將應用拆分為多層如資料庫、應用伺服器等

AWS ParallelCluster

  • 利用簡單設定檔即可快速部署與管理高效能運算（HPC）叢集
  • 完全控制底層資源，免費使用（資源另計）
  • 可結合 Elastic Fabric Adapter（EFA）取得低延遲網路

Terraform（業界標準 IaC 工具）

核心特性

  • 多雲支持：支援 AWS、Azure、Google Cloud、本地資源等多個雲平台
  • 開源免費：由 HashiCorp 開發並開源
  • 宣告式語言：使用 HCL（HashiCorp Configuration Language）
  • 業界標準：廣泛應用於企業和開源社群
  • 狀態管理：透過 State File 追蹤資源狀態
  • Plan 機制：執行前預覽變更

基本概念

概念說明
Provider定義與哪個雲平台互動（如 aws、azurerm）
Resource定義要建立的基礎設施元素
Variable輸入變數，增加可重用性
Output輸出值，供其他配置參照
Module可重用的 Terraform 程式碼集合
State File記錄實際部署的資源狀態
Backend儲存 State File 的位置（本地、S3、Terraform Cloud）

Terraform 生命週期

1. terraform init     # 初始化，下載 Provider
2. terraform plan     # 預覽變更（重要！）
3. terraform apply    # 執行變更
4. terraform destroy  # 刪除資源

基本語法範例

# Provider 配置
provider "aws" {
  region = "us-east-1"
}

# 定義變數
variable "instance_type" {
  type        = string
  default     = "t3.micro"
  description = "EC2 Instance Type"
}

# 建立資源
resource "aws_instance" "web_server" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = var.instance_type

  tags = {
    Name = "WebServer"
  }
}

# 輸出值
output "public_ip" {
  value       = aws_instance.web_server.public_ip
  description = "Public IP of the EC2 instance"
}

Terraform State Management（狀態管理）

State File 重要性

  • 記錄 Terraform 管理的資源現況
  • 用於確定何時需要建立/更新/刪除資源
  • 敏感：包含密碼、金鑰等敏感資訊

State 儲存方式（Backend）

Backend說明用途
Local本地文件系統個人開發
S3AWS S3 Bucket團隊協作，支援遠程鎖定
Terraform CloudHashiCorp 管理企業級，含版本控制、權限管理
Consul分散式儲存高可用性

遠程 State 最佳實踐

terraform {
  backend "s3" {
    bucket         = "my-terraform-state"
    key            = "prod/terraform.tfstate"
    region         = "us-east-1"
    encrypt        = true              # 啟用加密
    dynamodb_table = "terraform-locks" # 狀態鎖定
  }
}

Terraform Module（模組化）

概念

  • 將相關配置組織為可重用的模組
  • 類似 CloudFormation 的 Nested Stacks

模組結構

my-module/
├── main.tf          # 主要資源定義
├── variables.tf     # 輸入變數
├── outputs.tf       # 輸出值
└── terraform.tfvars # 變數賦值

使用模組

module "vpc" {
  source = "./modules/vpc"
  
  vpc_cidr = "10.0.0.0/16"
  name     = "my-vpc"
}

# 參照模組輸出
resource "aws_security_group" "app" {
  vpc_id = module.vpc.vpc_id
}

Terraform 進階特性

Workspace（工作區）

  • 在同一配置下管理多個環境
  • 使用不同的 State File
terraform workspace new dev
terraform workspace select dev
terraform workspace show  # 顯示當前工作區

Provisioner（配置器）

  • 在資源建立後執行操作
  • 應慎用，通常用 User Data 或其他方式較佳
resource "aws_instance" "web" {
  # ...
  provisioner "remote-exec" {
    inline = [
      "sudo yum install -y httpd",
      "sudo systemctl start httpd"
    ]
  }
}

Data Sources（資料來源）

  • 查詢現有資源信息
  • 不建立新資源，僅讀取
data "aws_ami" "ubuntu" {
  most_recent = true
  
  filter {
    name   = "name"
    values = ["ubuntu/images/hvm-ssd/ubuntu-focal-20.04-amd64-server-*"]
  }
}

resource "aws_instance" "web" {
  ami = data.aws_ami.ubuntu.id
}

Terraform 工作流程最佳實踐

  1. Version Control：將 .tf 文件存放在 Git 中
  2. Remote State：使用 S3 + DynamoDB 管理狀態
  3. Pre-commit Hooks：自動格式化和驗證
  4. CI/CD 整合：GitHub Actions、GitLab CI 自動執行 plan/apply
  5. 環境隔離：使用 Workspace 或不同目錄管理開發/測試/生產環境
  6. 安全敏感資訊：使用 aws_secretsmanager 存放密鑰，不提交到 Git

Terraform 常用命令

命令說明
terraform init初始化 Terraform 工作目錄
terraform validate驗證配置文件語法
terraform plan預覽將執行的變更
terraform apply執行變更
terraform destroy刪除所有資源
terraform state list列出狀態中的資源
terraform state show <resource>顯示特定資源詳情
terraform import <resource> <id>匯入現有資源到狀態
terraform refresh更新狀態以匹配現實
terraform fmt格式化配置文件

CloudFormation 架構深入解析

核心概念

  • 範本（Template）：定義資源的 YAML/JSON 檔案
  • 堆疊（Stack）：CloudFormation 建立的資源集合
  • 變更集（Change Set）：更新前預覽變更

範本結構

區段必要說明
AWSTemplateFormatVersion否範本版本
Description否範本描述
Metadata否額外資訊
Parameters否輸入參數
Mappings否鍵值對照
Conditions否條件邏輯
Transform否SAM 或 Macros
Resources是AWS 資源定義
Outputs否輸出值

Parameters（參數）詳解

概念

  • 允許使用者在建立 Stack 時輸入值
  • 增加範本的可重用性

參數類型

類型說明
String字串
Number數字
CommaDelimitedList逗號分隔清單
AWS::SSM::Parameter::ValueParameter Store 值
List數字清單
AWS-SpecificAWS 特定類型（如 VPC ID）

參數屬性

Parameters:
  InstanceType:
    Type: String
    Default: t3.micro
    AllowedValues:
      - t3.micro
      - t3.small
      - t3.medium
    Description: EC2 Instance Type

偽參數

偽參數說明
AWS::Region目前區域
AWS::AccountId帳號 ID
AWS::StackName堆疊名稱
AWS::StackId堆疊 ID
AWS::NoValue移除屬性

Intrinsic Functions（內建函數）

常用函數

函數用途範例
!Ref參照參數或資源!Ref MyEC2
!GetAtt取得資源屬性!GetAtt MyEC2.PublicIp
!Sub字串替換!Sub "arn:aws:s3:::${BucketName}"
!Join連接字串!Join ["-", [a, b, c]]
!Select選取清單元素!Select [0, !Ref MyList]
!Split分割字串!Split [",", !Ref MyString]
!If條件判斷!If [CreateProd, m5.large, t3.micro]

Fn::GetAZs

  • 取得區域的可用區域清單
  • 常與 !Select 結合使用
!Select [0, !GetAZs '']  # 第一個 AZ

Mappings（對照表）

概念

  • 硬編碼的鍵值對照
  • 用於根據條件選擇值

使用案例

Mappings:
  RegionAMI:
    us-east-1:
      HVM64: ami-12345678
    us-west-2:
      HVM64: ami-87654321
      
Resources:
  MyEC2:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: !FindInMap [RegionAMI, !Ref "AWS::Region", HVM64]

Outputs（輸出）

概念

  • 宣告可被其他 Stack 參照的值
  • 用於跨 Stack 參照

匯出語法

Outputs:
  VPCId:
    Description: VPC ID
    Value: !Ref MyVPC
    Export:
      Name: !Sub "${AWS::StackName}-VPCID"

匯入語法

!ImportValue <ExportName>

Conditions（條件）

概念

  • 根據條件決定是否建立資源
  • 根據參數或偽參數決定

條件函數

函數說明
!Equals相等判斷
!And且
!Or或
!Not非
!If條件判斷

範例

Conditions:
  CreateProdResources: !Equals [!Ref EnvType, prod]
  
Resources:
  ProdOnlyBucket:
    Type: AWS::S3::Bucket
    Condition: CreateProdResources

DependsOn

概念

  • 控制資源建立順序
  • CloudFormation 通常自動處理相依性
