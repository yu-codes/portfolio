概述

  • 提供整個帳號內所有操作的稽核紀錄與事件歷史
  • 記錄來源：使用者、服務、CLI、SDK
  • 自動預設啟用全區域記錄

事件類型

類型說明預設
Management EventsAPI 呼叫、登入等管理事件啟用
Data EventsS3 物件、Lambda 呼叫等資料事件停用
Insights Events異常活動偵測停用

Management Events

  • 建立、修改、刪除資源
  • 登入 Console
  • 設定安全群組

Data Events

  • S3 物件層級操作（GetObject、PutObject）
  • Lambda 函數呼叫
  • DynamoDB 表操作

輸出目的地

目的地用途
S3長期儲存、合規
CloudWatch Logs即時監控、警示

Trail 類型

類型範圍
單一區域僅記錄該區域事件
所有區域記錄所有區域事件（推薦）
組織 Trail整個 Organizations 的事件

日誌檔案完整性

日誌驗證

  • 使用 SHA-256 雜湊
  • 每小時產生摘要檔案
  • 可驗證日誌是否被竄改

S3 保護

  • 使用 S3 Bucket Policy 保護
  • 啟用 S3 版本控制
  • 使用 MFA Delete

CloudTrail Insights

  • 偵測異常 API 活動
  • 基於正常基準線分析
  • 產生 Insights 事件

常見使用案例

1. 資源變更追蹤

  • 追蹤誰在何時修改了什麼
  • 例如：追蹤 Security Group 變更

2. 合規稽核

  • 保留 API 呼叫記錄
  • 符合合規要求

3. 安全分析

  • 偵測未授權存取
  • 識別異常行為

4. 問題除錯

  • 追蹤資源是否被誤刪除或異常變更
  • 分析 API 呼叫序列

CloudTrail 進階功能

CloudTrail Lake

  • 不可變的事件儲存
  • SQL 查詢事件資料
  • 長期保留（最長 7 年）

CloudTrail Insights

  • 自動偵測異常 API 活動
  • 基於歷史基準線分析
  • 產生 Insights 事件

與 CloudWatch 整合

即時警示

  1. CloudTrail → CloudWatch Logs
  2. 設定 Metric Filter
  3. 建立 CloudWatch Alarm
  4. 觸發 SNS 通知

範例：Root 帳號登入警示

  • 建立 Metric Filter 過濾 Root 登入事件
  • 設定警示通知安全團隊

CloudTrail vs CloudWatch

項目CloudTrailCloudWatch
用途稽核 API 活動監控效能指標
資料事件（誰做了什麼）指標（資源狀態）
保留90 天（預設）或 S3依設定
適用合規、安全營運監控

最佳實踐

  1. 啟用所有區域 Trail：確保全面記錄
  2. 輸出至 S3：長期保留和合規
  3. 整合 CloudWatch Logs：即時監控
  4. 啟用日誌驗證：確保日誌完整性
  5. 保護 S3 Bucket：防止日誌被竄改
  6. 考慮 Insights：偵測異常活動
  7. 使用 CloudTrail Lake：進階事件分析

CloudTrail 架構深入解析

核心概念

  • 記錄 AWS 帳號的 API 呼叫/活動
  • 90 天歷史（預設，免費）
  • 需要 Trail 才能長期保留
  • 儲存在 S3 或 CloudWatch Logs

事件類型

類型說明預設
Management Events控制平面操作啟用
Data Events資料平面操作停用

Management Events

  • 建立 EC2 實例
  • 建立 VPC
  • 登入 Console
  • 免費包含

Data Events

  • S3 物件層級操作
  • Lambda 函數調用
  • 需額外付費

Trail 配置

Trail 類型

類型範圍
單一區域僅記錄一個區域
所有區域記錄所有區域（推薦）
組織 Trail所有成員帳號

全域服務事件

  • IAM、STS、CloudFront 等
  • 只記錄在 us-east-1
  • 需在 Trail 中啟用

CloudTrail 安全性

日誌完整性

  • Log File Validation：SHA-256 雜湊
  • 摘要檔案每小時產生
  • 可驗證日誌是否被竄改

S3 保護

措施說明
Bucket Policy限制存取
SSE-S3/SSE-KMS加密日誌
版本控制保留歷史
MFA Delete防止誤刪
Object Lock不可變性

CloudTrail 進階功能

CloudTrail Lake

  • 集中式事件儲存
  • 使用 SQL 查詢
  • 最長保留 7 年
  • 跨組織查詢

CloudTrail Insights

功能說明
基準線建立正常 API 活動模式
異常偵測偵測偏離基準線的活動
事件產生產生 Insights 事件

常見使用場景

1. 安全調查

誰刪除了某個 EC2 實例？

CloudTrail → 搜尋 TerminateInstances API
           → 找到使用者、時間、IP

2. 合規證明

證明符合安全要求：

  • 所有 API 操作都有記錄
  • 日誌無法被竄改
  • 長期保留

3. 即時警示

CloudTrail → CloudWatch Logs → Metric Filter → Alarm → SNS

範例：Root 帳號登入警示
