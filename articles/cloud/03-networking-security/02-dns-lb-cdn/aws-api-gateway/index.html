概述

  • Serverless API 管理服務
  • 支援 HTTP、WebSocket、REST 類型
  • 提供限制頻率（throttle limit）與快取機制

API 類型

類型說明適用場景
REST API功能完整的 RESTful API需要進階功能
HTTP API輕量、低延遲簡單 HTTP 代理
WebSocket API雙向即時通訊聊天、遊戲、即時更新

REST API vs HTTP API

項目REST APIHTTP API
成本較高較低（約 70% 便宜）
延遲較高較低
功能完整（快取、WAF 等）基本
驗證IAM、Lambda、CognitoIAM、Lambda、JWT

驗證方式

方式說明
IAM Policy使用 IAM 角色和政策
Lambda Authorizer自訂驗證邏輯
Cognito User Pool使用 Cognito 進行身份驗證

Lambda Authorizer

  • 自訂驗證邏輯
  • 可整合現有身份系統
  • 支援 Token 和 Request 兩種類型

Cognito User Pool

  • 託管用戶身份
  • 支援社群登入
  • 回傳 JWT Token

整合類型

類型說明
Lambda呼叫 Lambda 函數
HTTP代理到 HTTP 端點
AWS Service直接呼叫 AWS 服務
VPC Link存取 VPC 內部資源
Mock模擬回應（測試用）

部署與階段

階段（Stages）

  • 代表 API 的不同版本
  • 例如：dev、staging、prod
  • 每個階段有獨立的 URL

階段變數

  • 類似環境變數
  • 可在不同階段指向不同後端

Canary 部署

  • 漸進式部署
  • 將部分流量導向新版本

限流與配額

帳號層級限制

限制值
預設請求限制10,000 請求/秒
預設並發限制5,000

使用計畫（Usage Plans）

  • 為 API 金鑰設定配額
  • 限制每日/每月請求數
  • 限流（Throttling）設定

快取

  • 可快取端點回應
  • 減少後端呼叫
  • 可配置 TTL（0-3600 秒）
  • 可針對特定方法啟用

安全性

WAF 整合

  • 保護 API 免受常見攻擊
  • SQL Injection、XSS 防護
  • 僅 REST API 支援

CORS

  • 跨來源資源共享
  • 需在 API Gateway 設定

資源政策

  • 控制誰可以存取 API
  • 支援 IP 白名單/黑名單
  • 支援 VPC 端點存取

Private API

  • 僅能從 VPC 內存取
  • 使用 VPC Interface Endpoint
  • 透過資源政策控制存取

Amazon Cognito 整合

Cognito User Pools

  • 提供使用者目錄進行註冊和登入
  • 管理使用者設定檔
  • 發行 JWT Token

驗證方式

方式說明
帳號密碼傳統驗證方式
社群登入Facebook、Google、Amazon、Apple
MFASMS 或 TOTP
SAML/OIDC企業身份聯合

Cognito Identity Pools（Federated Identities）

  • 授予使用者臨時 AWS 認證
  • 可直接存取 AWS 服務（如 S3、DynamoDB）
  • 支援未經驗證的訪客存取

使用者遷移

  • 支援從現有使用者目錄遷移
  • Lambda 觸發器處理遷移邏輯

監控

CloudWatch 指標

指標說明

Latency延遲
IntegrationLatency後端延遲
4XXError客戶端錯誤
5XXError伺服器錯誤
CacheHitCount快取命中
CacheMissCount快取未命中

X-Ray 追蹤

  • 分散式追蹤
  • 效能分析

與其他服務整合

常見架構

Client → API Gateway → Lambda → DynamoDB
                     → SQS
                     → Step Functions

VPC Link

  • 存取 VPC 內的 NLB/ALB
  • 私有 API 整合

最佳實踐

  1. 使用適當的 API 類型：簡單場景用 HTTP API
  2. 啟用快取：減少後端負載
  3. 實施限流：保護後端服務
  4. 使用階段：分離開發和生產環境
  5. 整合 WAF：保護 API 安全
  6. 監控延遲：追蹤效能問題
  7. 使用 Lambda Authorizer：自訂驗證需求
  8. 使用 Cognito：簡化使用者管理
  9. 考慮 Private API：內部服務使用

Amazon CloudFront

概述

  • AWS 的全球內容傳遞網路（CDN）
  • 使用邊緣位置快取內容
  • 降低延遲、提高效能

CloudFront 架構詳解

核心概念

概念說明
Edge Location快取內容的位置（全球 400+）
Regional Edge Cache區域邊緣快取（較大）
Origin內容來源（S3、ALB、自訂）
DistributionCloudFront 配置單位

請求流程

使用者 → 最近的 Edge → Regional Edge Cache → Origin
              ↓
         快取命中 → 直接返回

Origin 類型

S3 Origin

功能說明
靜態內容網站、圖片、影片
OAI/OAC限制 S3 只能透過 CloudFront 存取
跨區域複製結合 S3 Replication

Custom Origin

類型說明
ALB應用程式負載平衡器
EC2需要公開 IP
自訂 HTTP任何 HTTP 伺服器
MediaStore影片串流

Origin Access Identity (OAI)

概念

  • 限制 S3 只能透過 CloudFront 存取
  • S3 Bucket 不需公開

配置

  1. 建立 OAI
  2. 在 CloudFront 分發中配置
  3. 更新 S3 Bucket Policy

Origin Access Control (OAC)

  • OAI 的新版替代方案
  • 支援更多功能
  • 建議使用 OAC

Behaviors（行為）

概念

  • 定義 CloudFront 如何處理請求
  • 基於路徑模式匹配
  • 每個 Distribution 可有多個 Behavior

設定項目

設定說明
Path Pattern路徑匹配（如 /images/*）
Origin請求轉發的目標
ProtocolHTTP/HTTPS 設定
Cache Policy快取行為
TTL快取時間

TTL 與快取

TTL 類型

TTL說明
Minimum TTL最小快取時間
Maximum TTL最大快取時間
Default TTL預設快取時間

Cache-Control Header

  • Origin 可設定 Cache-Control
  • CloudFront 遵守 Header 設定
  • 在 Min/Max TTL 範圍內

快取失效（Invalidation）

  • 手動清除快取
  • 使用路徑模式（如 /* 或 /images/*）
  • 有費用，考慮使用版本化 URL

SSL/TLS 與 ACM

憑證來源

來源區域
ACM必須在 us-east-1
IAM較舊方式

SNI（Server Name Indication）

  • 允許多個憑證在同一 IP
  • 預設免費
  • 舊瀏覽器需要專用 IP（費用較高）

Viewer Protocol Policy

選項說明
HTTP and HTTPS兩者皆可
Redirect HTTP to HTTPS重導向（推薦）
HTTPS Only僅 HTTPS

地理限制

功能

  • 允許或封鎖特定國家
  • 基於 IP 的地理位置

使用案例

  • 內容授權限制
  • 合規要求
  • 成本控制

Signed URLs 與 Signed Cookies

使用場景

  • 限制內容存取
  • 付費內容
  • 敏感資料

Signed URL vs Signed Cookie

項目Signed URLSigned Cookie
粒度單一檔案多個檔案
適用個別內容整個網站區域
實作URL 參數Cookie

信任金鑰

類型說明
CloudFront Key Pair舊方式（Root 帳號）
Trusted Key Groups新方式（IAM 管理）

Lambda@Edge

觸發點用途

Origin Request轉發到 Origin 前
Origin ResponseOrigin 回應後
Viewer Response返回給使用者前

Lambda@Edge vs CloudFront Functions

項目Lambda@EdgeCloudFront Functions
執行時期Node.js/PythonJavaScript
執行時間最長 30 秒< 1ms
記憶體最大 10GB2MB
網路存取是否
觸發點全部 4 個Viewer 2 個
成本較高較低
適用複雜邏輯簡單轉換

AWS Global Accelerator

概述

  • 非 CDN，是網路最佳化服務
  • 使用 AWS 骨幹網路
  • 提供靜態 Anycast IP

CloudFront vs Global Accelerator

項目CloudFrontGlobal Accelerator
用途內容快取網路加速
快取是否
IPDNS 名稱靜態 IP
協定HTTP/HTTPSTCP/UDP
適用靜態/動態內容即時應用、遊戲

Global Accelerator 使用案例

場景說明
需要靜態 IP防火牆白名單
非 HTTPTCP/UDP 應用
即時性遊戲、交易
故障轉移快速故障轉移

CloudFront 安全性

WAF 整合

  • 保護 Distribution
  • SQL Injection、XSS 防護
  • 速率限制

Field-Level Encryption

  • 在邊緣加密敏感欄位
  • 僅後端可解密
  • 信用卡等敏感資料

Origin Shield

  • 額外的快取層
  • 減少 Origin 負載
  • 改善快取命中率
