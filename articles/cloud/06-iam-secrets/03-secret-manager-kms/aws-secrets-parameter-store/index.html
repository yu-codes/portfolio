AWS Secrets Manager

  • 管理密碼/金鑰/API Token 的儲存與自動輪替
  • 支援 RDS（MySQL/PostgreSQL/Aurora） 的原生密碼輪替
  • 其他服務可透過 Lambda 自訂輪替流程

主要功能

功能說明
秘密儲存安全儲存敏感資訊
自動輪替定期自動更換密碼
存取控制透過 IAM 政策控制存取
加密使用 KMS 加密秘密
審計透過 CloudTrail 記錄存取

支援的自動輪替

  • Amazon RDS（MySQL、PostgreSQL、Aurora）
  • Amazon Redshift
  • Amazon DocumentDB
  • 其他服務（透過 Lambda 自訂）

AWS Systems Manager Parameter Store

  • 集中管理密碼、環境變數（支援 KMS 加密）
  • 屬於 AWS Systems Manager（SSM）的一部分

Parameter Store 類型

類型說明大小限制
Standard免費，基本功能4 KB
Advanced付費，進階功能8 KB

參數類型

類型說明
String一般字串
StringList逗號分隔的字串列表
SecureString使用 KMS 加密的字串

AWS Systems Manager 其他功能

Run Command

  • 遠端對多台 EC2 批次下指令

Session Manager

  • 替代 Bastion，安全登入 Private Subnet 中的 EC2
  • 不需要開放 SSH 端口
  • 提供完整的操作審計記錄

使用場景建議

使用 Secrets Manager

  • 資料庫密碼（特別是 RDS）
  • 需要自動輪替的敏感資訊
  • API 金鑰和存取令牌

使用 Parameter Store

  • 應用程式設定參數
  • 環境變數
  • 不需要自動輪替的設定值
  • 成本敏感的使用場景

最佳實踐

  1. 永遠使用 SecureString：敏感資訊應加密儲存
  2. 實施最小權限：只授予必要的存取權限
  3. 啟用審計：透過 CloudTrail 追蹤所有存取
  4. 使用版本管理：追蹤設定變更歷史
  5. 自動輪替：定期更換密碼和金鑰
