Core Concepts

What is Reactivity?

Reactivity means that when data changes, the related UI automatically synchronizes. Vue 3 uses ES6 Proxy to intercept data reads (get) and writes (set), automatically tracking dependencies and triggering updates.

Vue 2 vs Vue 3 Reactivity

FeatureVue 2 (Object.defineProperty)Vue 3 (Proxy)
New property detection❌ Requires Vue.set()✅ Auto-detected
Array index watching❌ Cannot watch directly✅ Full support
Map/Set support❌ Not supported✅ Native support
PerformanceTraverses all properties on initLazy interception, on-demand tracking

Architecture Overview

[User Interaction] → [Modify reactive data] → [Proxy set intercept]
       ↓
[trigger()] → [Notify all dependent effects]
       ↓
[Re-execute render / computed / watch]
       ↓
[Virtual DOM diff → Update real DOM]

Dependency Collection and Trigger Flow

  1. track() — Called in Proxy’s get interceptor, records current effect (e.g., render function) as a dependency
  2. trigger() — Called in Proxy’s set interceptor, notifies all dependent effects to re-execute
  3. effect() — Wraps side-effect functions so they can be tracked by the dependency system

ref vs reactive Complete Comparison

ref

import { ref } from 'vue'

const count = ref(0)        // Create ref
console.log(count.value)    // Read requires .value
count.value++               // Modify requires .value

// In templates, auto-unwrapped - no .value needed
// <template>{{ count }}</template>

Use cases: Primitive types (number, string, boolean), scenarios requiring reassignment.

reactive

import { reactive } from 'vue'

const state = reactive({
  count: 0,
  user: { name: 'Yu' }
})

state.count++              // Direct access, no .value
state.user.name = 'Test'   // Deep nesting is also reactive

Use cases: Objects, nested structures, form data.

Selection Guide

ScenarioRecommendedReason
Primitives (number/string/boolean)refreactive can’t wrap primitives
Objects/nested structuresreactive or refreactive access is more intuitive
Need to replace entire objectrefreactive can’t be reassigned
Composable return valuesrefDestructuring won’t lose reactivity

computed

import { ref, computed } from 'vue'

const firstName = ref('Yu')
const lastName = ref('Han')

// Read-only computed
const fullName = computed(() => `${firstName.value} ${lastName.value}`)

// Writable computed
const fullNameWritable = computed({
  get: () => `${firstName.value} ${lastName.value}`,
  set: (val) => {
    const [first, last] = val.split(' ')
    firstName.value = first
    lastName.value = last
  }
})

Key characteristics:

  • Caching: Only recalculates when dependencies change
  • Lazy evaluation: Won’t execute until read
  • vs methods: Methods execute on every render, computed has caching advantage

watch and watchEffect

watch — Precise Watching

import { ref, watch } from 'vue'

const count = ref(0)

// Watch single ref
watch(count, (newVal, oldVal) => {
  console.log(`count: ${oldVal} → ${newVal}`)
})

// Watch multiple sources
watch([count, anotherRef], ([newCount, newOther], [oldCount, oldOther]) => {
  // ...
})

// Deep watch object
watch(
  () => state.nested,
  (newVal) => { /* ... */ },
  { deep: true, immediate: true }
)

watchEffect — Auto-tracking

import { ref, watchEffect } from 'vue'

const count = ref(0)

// Automatically tracks all reactive data used
const stop = watchEffect(() => {
  console.log(`count is: ${count.value}`)
})

// Stop watching
stop()

watch vs watchEffect Selection

FeaturewatchwatchEffect
Specify watch source✅ Yes❌ Auto-tracked
Get old value✅ Yes❌ No
Initial executionNeeds immediate: trueExecutes immediately by default
Use casePrecise control, need old valueEffect sync, simple scenarios

toRef / toRefs — Reactive Destructuring

import { reactive, toRef, toRefs } from 'vue'

const state = reactive({ count: 0, name: 'Yu' })

// Wrong: destructuring breaks reactivity
const { count } = state  // count is not reactive

// Correct: use toRef / toRefs
const countRef = toRef(state, 'count')
const { count: countRef2, name } = toRefs(state)

Use Cases

Form Handling

const form = reactive({
  username: '',
  email: '',
  password: ''
})

const isValid = computed(() =>
  form.username.length > 0 &&
  form.email.includes('@') &&
  form.password.length >= 8
)

Composable Encapsulation

// useCounter.ts
export function useCounter(initial = 0) {
  const count = ref(initial)
  const doubled = computed(() => count.value * 2)
  const increment = () => count.value++
  const decrement = () => count.value--
  return { count, doubled, increment, decrement }
}
