分類與回歸指標

選擇正確的評估指標，和選擇正確的模型一樣重要。用錯指標，可能讓你優化了一個完全錯誤的方向。

分類指標

混淆矩陣：一切的基礎

二元分類的混淆矩陣包含四個值：

  • True Positive (TP)：預測正確的正樣本
  • True Negative (TN)：預測正確的負樣本
  • False Positive (FP)：把負樣本誤判為正（型一錯誤）
  • False Negative (FN)：把正樣本誤判為負（型二錯誤）

所有分類指標都可以從這四個值推導出來。

準確率（Accuracy）

$Accuracy = \frac{TP + TN}{TP + TN + FP + FN}$

最直覺的指標，但在類別不平衡時會嚴重誤導。例如 99% 的樣本是負類，一個永遠預測「負」的模型就有 99% 的準確率。

精確率（Precision）

$Precision = \frac{TP}{TP + FP}$

在所有被預測為正的樣本中，有多少真的是正的？

重要場景：垃圾郵件偵測——你不希望正常郵件被誤判為垃圾郵件。高 Precision = 低誤報率。

召回率（Recall）

$Recall = \frac{TP}{TP + FN}$

在所有真正的正樣本中，有多少被正確找到了？

重要場景：癌症篩檢——你不希望漏掉真正的癌症患者。高 Recall = 低漏報率。

F1 Score

$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$

Precision 和 Recall 的調和平均。在兩者需要平衡時使用。

AUC-ROC

ROC 曲線描繪了在不同閾值下，True Positive Rate（Recall）和 False Positive Rate 的關係。

AUC 是 ROC 曲線下的面積。它的優點是不受閾值選擇影響——衡量的是模型整體的排序能力。

AUC = 0.5 表示隨機猜測，AUC = 1.0 表示完美分類。

PR-AUC

在類別極度不平衡時，AUC-ROC 可能過於樂觀。PR-AUC（Precision-Recall AUC）更適合這種場景。

回歸指標

MSE / RMSE

MSE 是預測誤差平方的平均值。RMSE 是 MSE 的平方根，單位與目標變數相同，更直覺。

MAE

預測誤差絕對值的平均。比 MSE 對離群值更穩健。

R² Score

$R^2 = 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}$

衡量模型解釋了多少比例的變異。R² = 1 完美預測，R² = 0 等於用平均值預測。

注意：R² 可以是負的——代表模型比用平均值預測還差。

MAPE（平均絕對百分比誤差）

$MAPE = \frac{1}{n}\sum|\frac{y_i - \hat{y}_i}{y_i}|$

用百分比衡量誤差，讓不同尺度的問題可以比較。但當真實值接近零時會爆炸。

指標選擇的決策框架

問題類型推薦指標原因
平衡二元分類Accuracy, F1直覺且合理
不平衡二元分類PR-AUC, F1避免被多數類主導
多類別分類Macro F1對每個類別同等重視
回歸（一般）RMSE對大誤差敏感
回歸（有離群值）MAE更穩健
排序AUC-ROC, NDCG重點在排序而非絕對值

業務指標 vs 模型指標

模型指標（如 F1、AUC）是技術人員看的。業務指標（如營收增長、用戶留存率）是老闆看的。

兩者之間需要建立對應關係。例如：「推薦系統的 AUC 從 0.85 提升到 0.90，帶來了 5% 的點擊率提升，預估每月增加 X 萬營收。」

如果模型指標改善但業務指標沒有改善，說明你可能在優化錯誤的目標。

實務心法

  1. 永遠先搞清楚業務在意什麼，再選技術指標
  2. 類別不平衡時絕對不要用 Accuracy
  3. 報告多個指標，不要只看一個——每個指標都只反映了部分真相
  4. 閾值的選擇是業務決策，不是技術決策
