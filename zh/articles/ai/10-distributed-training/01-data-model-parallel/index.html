資料平行與模型平行

突破單機限制：分散式訓練的兩大基本策略

當你的模型太大、資料太多、或訓練時間太長時，單一 GPU 已經不夠用了。分散式訓練把工作分散到多個 GPU 甚至多台機器上。

資料平行（Data Parallelism）

核心思想

每個 GPU 持有完整的模型副本，但處理不同的資料批次。前向傳播後，收集所有 GPU 的梯度做平均，再更新所有副本的參數。

工作流程

  1. 把一個大批次分成 N 個小批次（N = GPU 數量）
  2. 每個 GPU 用自己的小批次做前向傳播和反向傳播
  3. 所有 GPU 同步梯度（AllReduce 操作）
  4. 每個 GPU 用平均梯度更新本地參數

AllReduce

AllReduce 是分散式訓練的核心通訊操作。它把所有 GPU 上的梯度加總後，把結果分發回每個 GPU。

高效的 AllReduce 實作：

  • Ring AllReduce：每個 GPU 只與鄰居通訊，通訊量不隨 GPU 數增加
  • NCCL：NVIDIA 的通訊庫，為 GPU 間通訊高度優化

大批次訓練的挑戰

資料平行的總批次大小 = 單 GPU 批次 × GPU 數。但大批次可能影響收斂：

  • 學習率需要隨批次大小線性增大
  • 但太大的學習率會導致訓練不穩定
  • 需要 warmup 來穩定初期訓練
  • 通常有一個「最佳批次大小」的上限

模型平行（Model Parallelism）

核心思想

當模型太大，一個 GPU 放不下時，把模型切成多段放在不同的 GPU 上。

樸素的模型平行

按層切割：前幾層在 GPU 0，後幾層在 GPU 1。

致命問題：氣泡問題（Bubble Problem）——GPU 0 計算完把結果送給 GPU 1 時，GPU 0 就閒置了。GPU 利用率極低。

管線平行（Pipeline Parallelism）

解決氣泡問題的方法：把一個大批次切成多個微批次（micro-batch），讓不同的微批次交錯在不同的管線階段上。

GPipe 和 PipeDream 是兩個重要的管線平行方案。

張量平行（Tensor Parallelism）

把單個層的計算分佈到多個 GPU 上。例如，一個大的矩陣乘法，把矩陣切成幾塊，每個 GPU 算一塊。

Megatron-LM 就是用張量平行來訓練超大型語言模型。

混合平行

實際上，大規模訓練通常是多種平行策略的組合：

  • 機器內：張量平行（GPU 之間頻寬最高）
  • 機器間：管線平行 + 資料平行

例如，訓練 GPT-3 使用了：

  • 8 路張量平行（一台機器內的 8 張 GPU）
  • 64 路管線平行（64 個管線階段）
  • 8 路資料平行

ZeRO（Zero Redundancy Optimizer）

微軟提出的記憶體優化策略，消除資料平行中的冗餘記憶體：

ZeRO Stage 1：分片優化器狀態 ZeRO Stage 2：再分片梯度 ZeRO Stage 3：再分片模型參數

ZeRO-3 讓每個 GPU 只儲存模型的 1/N（N = GPU 數），大幅降低記憶體需求。

PyTorch 的 FSDP 就是 ZeRO-3 的實作。

通訊效率

通訊瓶頸

多 GPU 訓練的效能往往受限於 GPU 之間的通訊速度。

互聯方式頻寬
PCIe Gen432 GB/s
NVLink 3.0600 GB/s
InfiniBand HDR200 Gb/s

通訊優化技巧

  • 梯度壓縮：傳輸前壓縮梯度，犧牲少量精度換取通訊效率
  • 梯度累積：累積多步梯度再同步，減少通訊次數
  • 計算-通訊重疊：在通訊的同時進行計算

實務心法

  1. 小模型（<10B 參數）用資料平行就好——簡單有效
  2. 大模型先考慮 ZeRO/FSDP——實作簡單且效果好
  3. 超大模型需要混合平行——這是專業領域，需要深入了解
  4. 通訊效率決定了多 GPU 訓練的擴展性——關注互聯硬體
