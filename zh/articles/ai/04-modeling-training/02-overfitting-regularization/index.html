過擬合與正則化

深入理解過擬合現象：從診斷到多種正則化策略的實務應用

過擬合是 ML 實務中最頻繁遇到的問題。你的模型在訓練集上表現 99%，但在新資料上只有 70%——這就是過擬合。理解它的成因和解決方案，是成為成熟 ML 工程師的必經之路。

過擬合的本質

過擬合的本質是模型的容量（capacity）超過了資料所能支撐的範圍。模型有太多的「自由度」，它不僅學到了資料中的規律，也學到了噪聲。

偏差-方差分解

模型的預測誤差可以分解為三個部分：

偏差（Bias）：模型假設與真實關係之間的差距。高偏差 = 模型太簡單。

方差（Variance）：模型對訓練資料的敏感度。高方差 = 模型太複雜。

不可約誤差：資料本身的噪聲，無法消除。

過擬合 = 低偏差 + 高方差。欠擬合 = 高偏差 + 低方差。

過擬合的診斷信號

  • 訓練 loss 持續下降，驗證 loss 開始上升
  • 訓練準確率遠高於驗證準確率（差距 > 10%）
  • 模型在不同的訓練集上表現差異很大

正則化策略大全

資料層面

增加資料量：最直接有效的方法。更多資料意味著噪聲的影響會被稀釋。

資料增強（Data Augmentation）：

  • 圖片：旋轉、翻轉、裁切、色彩調整、Mixup、CutMix
  • 文字：同義詞替換、回譯（back translation）、隨機刪除
  • 表格：SMOTE（合成過採樣）

模型層面

降低模型複雜度：

  • 減少神經網路的層數或寬度
  • 減少決策樹的深度
  • 降低多項式的次數

Dropout：訓練時隨機關閉一定比例的神經元。常用比例 0.1-0.5。Dropout 迫使網路學習更穩健的特徵，因為它不能依賴任何單一神經元。

Batch Normalization：穩定每層的輸入分佈。雖然設計初衷是加速訓練，但也有正則化效果。

Weight Decay（權重衰減）：在每次參數更新時，將參數乘以一個略小於 1 的係數。等價於 L2 正則化。

訓練策略層面

Early Stopping：監控驗證集的 loss，當它不再改善時停止訓練。需要設定耐心（patience）參數。

學習率排程：逐步降低學習率，讓模型在訓練後期做更精細的調整，而非大幅跳動。

Label Smoothing：把硬標籤（0 或 1）轉為軟標籤（0.1 或 0.9）。防止模型過度自信。

集成方法

Bagging：訓練多個模型，取平均。隨機森林就是這個原理。

Stacking：用多個基礎模型的預測作為特徵，訓練一個元模型。

集成方法幾乎總是能減少過擬合，因為不同模型的錯誤傾向被平均掉了。

各模型的正則化最佳實踐

模型類型推薦正則化策略
線性模型L1/L2/Elastic Net
決策樹限制深度、最小葉子樣本數
XGBoostmax_depth + min_child_weight + subsample
神經網路Dropout + Weight Decay + Early Stopping
TransformerDropout + Label Smoothing + Weight Decay

實務心法

  1. 先確認是否真的過擬合——不是所有效能差都是過擬合的問題
  2. 增加資料量是最有效的正則化——如果可能的話
  3. 正則化的強度需要調參——用交叉驗證找到最佳值
  4. 多種正則化可以疊加使用——Dropout + Weight Decay + Early Stopping 是常見組合
