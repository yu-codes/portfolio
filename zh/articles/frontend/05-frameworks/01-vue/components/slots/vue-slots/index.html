Vue 3 Slots Deep Dive: Default, Named & Scoped Slots

Yu 最後更新: 2025-03-01

Slots are Vue’s content distribution mechanism that lets parent components customize child component rendering - a key technique for building reusable components.

Core Concepts

Default Slot

<!-- Child: BaseCard.vue -->
<div class="card">
  <slot>Default content</slot>
</div>

<!-- Parent -->
<BaseCard>
  <p>Custom content replaces default slot</p>
</BaseCard>

Named Slots

<!-- Child: PageLayout.vue -->
<header><slot name="header"></slot></header>
<main><slot></slot></main>
<footer><slot name="footer"></slot></footer>

<!-- Parent -->
<PageLayout>
  <template #header><h1>Title</h1></template>
  <p>Main content</p>
  <template #footer><p>Footer</p></template>
</PageLayout>

Scoped Slots

<!-- Child: ItemList.vue -->
<ul>
  <li v-for="item in items" :key="item.id">
    <slot :item="item" :index="index">{{ item.name }}</slot>
  </li>
</ul>

<!-- Parent: Customize each item's rendering -->
<ItemList :items="items">
  <template #default="{ item, index }">
    <span>{{ index + 1 }}. {{ item.name }} - {{ item.price }}</span>
  </template>
</ItemList>

Architecture Overview

Slot TypeUse CaseSyntax
DefaultBasic content distribution<slot>
NamedMulti-area content<slot name="x"> / #x
ScopedChild passes data for parent to render<slot :data="val">
DynamicProgrammatic slot selection#[dynamicName]
