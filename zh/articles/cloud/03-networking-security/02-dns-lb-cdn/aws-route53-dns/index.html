概述

  • AWS 託管的 DNS 服務，可註冊網域、建立 Hosted Zone
  • 支援公開與私有域名（Private Hosted Zone）
  • 100% 可用性 SLA

Hosted Zone 類型

類型說明
Public Hosted Zone管理公開網域的 DNS 記錄
Private Hosted Zone管理 VPC 內部的私有 DNS 記錄

Record 類型

類型說明
A將域名對應到 IPv4 地址
AAAA將域名對應到 IPv6 地址
CNAME將域名對應到另一個域名（不能用於根域名）
AliasAWS 專用，可對應到 AWS 資源（可用於根域名）
MX郵件伺服器記錄
TXT文字記錄（用於驗證）
NS名稱伺服器記錄

CNAME vs Alias

項目CNAMEAlias
根域名支援不支援支援
AWS 資源需要完整 DNS 名稱原生支援
費用需付費查詢免費（對 AWS 資源）
TTL可自訂由 AWS 管理

Routing Policy（路由策略）

策略說明使用場景
Simple單一記錄，隨機回傳多個值基本路由
Weighted依權重分配流量A/B 測試、漸進式部署
Latency導向延遲最低的區域全球應用
Failover主動-被動故障轉移災難復原
Geolocation依使用者地理位置路由在地化內容
Geoproximity依地理位置和偏移路由進階地理路由
Multivalue Answer多值回應並進行健康檢查簡單負載分散

DNS Failover

Active-Active

  • 所有資源都接收流量
  • 使用 Weighted 或 Latency 路由

Active-Passive

  • 主要資源處理流量，備援待命
  • 使用 Failover 路由策略

健康檢查

檢查類型

類型說明
Endpoint直接檢查端點健康狀態
Calculated基於其他健康檢查的計算結果
CloudWatch Alarm基於 CloudWatch 警示狀態

健康檢查參數

  • Protocol：HTTP / HTTPS / TCP
  • Interval：10 或 30 秒
  • Failure Threshold：連續失敗次數
  • String Matching：回應內容檢查

Traffic Flow

  • 視覺化的路由策略編輯器
  • 支援複雜的路由邏輯組合
  • 可建立可重用的策略

Route 53 Resolver

Resolver Endpoints

類型說明
Inbound允許本地 DNS 查詢 AWS VPC 中的資源
Outbound允許 VPC 中的資源查詢本地 DNS

DNSSEC（DNS 安全擴充）

  • 提供 DNS 回應的真實性和完整性驗證
  • 防止 DNS 欺騙和快取中毒攻擊
  • 支援 Route 53 公共託管區域

與其他服務整合

支援 Alias 的 AWS 資源

  • ELB（ALB / NLB / CLB）
  • CloudFront Distribution
  • API Gateway
  • S3 Website
  • Elastic Beanstalk
  • VPC Interface Endpoint
  • Global Accelerator

最佳實踐

  1. 使用 Alias 記錄：指向 AWS 資源時免費且支援根域名
  2. 設定健康檢查：確保流量導向健康的端點
  3. 實施故障轉移：使用 Failover 路由策略
  4. 考慮延遲路由：全球用戶使用 Latency 路由
  5. 使用 Private Hosted Zone：VPC 內部 DNS 解析
  6. 啟用 DNSSEC：加強 DNS 安全性

DNS 基礎知識

DNS 概述

  • DNS 是一種探索服務
  • 分散式資料庫
  • 將機器名稱轉換為人類可讀名稱（反之亦然）
  • 規模龐大，必須分散
  • Zone 檔案可被查詢

重要術語

術語說明
DNS Client您的裝置（筆電、手機、平板、PC 等）
Resolver您裝置上的軟體或代為查詢 DNS 的伺服器
ZoneDNS 資料庫的一部分（如 amazon.com）
ZonefileZone 的實體資料庫
Nameserver存放 Zonefile 的伺服器

DNS Root

  • DNS 的起點
  • 由 13 個 Root Server 託管（12 家大型公司和組織營運）
  • 每個 Root Server 可以是伺服器叢集
  • Root Hints：由供應商提供，指向 DNS Root Server 的清單
  • Root Zone 由 IANA（網際網路位址指派機構）營運

DNS 階層

Root Zone（IANA）→ TLD（.com, .org, .uk）→ 網域（amazon.com）

重要概念

  • Root hints：設定指向 Root Server 的 IP 和位址
  • Root Server：託管 DNS Root Zone
  • Root Zone：指向 TLD 授權伺服器
  • gTLD：通用頂級域名（.com, .org 等）
  • ccTLD：國家代碼頂級域名（.uk, .tw 等）
  • Registry：維護 TLD Zone 的組織
  • Registrar：與 TLD Zone 管理者有關係，允許網域註冊的組織

Route 53 基礎

R53 功能

  • 註冊網域
  • 託管 Zones（受管理的 Name Servers）
  • 全球伺服器，單一資料庫
  • 全球韌性

Hosted Zones

  • AWS 中的 Zone 檔案
  • 託管在 4 個受管理的 Name Servers
  • 可以是公開或私有（連結到 VPC）
  • 儲存記錄（Recordsets）

DNS 記錄類型詳解

Nameserver（NS）記錄

  • 允許 DNS 委派的記錄類型
  • 例如 .com Zone 中有 amazon.com 的多個 NS 記錄

A 和 AAAA 記錄

  • 將主機名稱對應到 IP
  • A：www → IPv4
  • AAAA：www → IPv6

CNAME 記錄

  • 主機對主機
  • ftp、mail、www（參考）→ A 記錄伺服器
  • 不能直接指向 IP，只能指向其他名稱

MX 記錄

  • 用於電子郵件
  • 較低的優先值 = 較高優先度
  • 例如：MX 10 mail、MX 20 mail.other.domain

TXT 記錄

  • 允許在網域新增任意文字
  • 用於證明網域擁有權

TTL（Time To Live）

  • 查詢結果在 Resolver 快取的時間（秒）
  • 授權：直接從授權伺服器取得的結果
  • 非授權：從 Resolver 快取取得的結果

CNAME vs Route 53 Alias

問題

  • “A” 記錄將名稱對應到 IP
  • CNAME 將名稱對應到另一個名稱
  • CNAME 無法用於裸域名/頂點域名（如 catagram.io）
  • 許多 AWS 服務使用 DNS 名稱（如 ELB）

Alias 解決方案

  • ALIAS 記錄將名稱對應到 AWS 資源
  • 可用於裸域名/頂點和一般記錄
  • 指向 AWS 資源的 ALIAS 請求免費
  • AWS 服務應預設選擇 ALIAS
  • 應與指向的記錄類型相同

支援 ALIAS 的資源

  • API Gateway
  • CloudFront
  • Elastic Beanstalk
  • ELB
  • Global Accelerator
  • S3

Route 53 健康檢查詳解

概述

  • 健康檢查與記錄分開，但被記錄使用
  • 健康檢查器位於全球各地
  • 每 30 秒檢查（每 10 秒需額外付費）
  • TCP、HTTP/HTTPS、HTTP/HTTPS + 字串匹配
  • 狀態：Healthy 或 Unhealthy
  • 類型：Endpoint、CloudWatch Alarm、Checks of Checks（Calculated）

路由策略詳解

Simple Routing（簡單路由）

  • 單一記錄，可有多個值
  • 隨機回傳所有值給客戶端
  • 不支援健康檢查

Failover Routing（故障轉移路由）

  • 建立兩個相同名稱和類型的記錄
  • 一個設為主要，一個設為次要
  • R53 知道兩個實例的健康狀態
  • 主要健康時回傳主要，失敗時回傳次要
  • 實現 Active-Passive 故障轉移

Multi Value Routing（多值路由）

  • 多個相同名稱的記錄，每個可有健康檢查
  • R53 回傳所有健康的記錄
  • 移除不健康的記錄
  • 簡單路由的改良版，改善可靠性
  • 適用於超過兩個記錄但不需加權路由複雜性的情況

Weighted Routing（加權路由）

  • 建立多個相同名稱的記錄，每個有權重值
  • 根據權重分配流量
  • 如果其中一個健康檢查失敗，會跳過
  • 適用於遷移、A/B 測試

Latency Routing（延遲路由）

  • 多個相同名稱和類型的記錄
  • 客戶端請求時，R53 知道請求來源區域
  • 回傳延遲最低的記錄
  • 適用於全球效能改善

Geolocation Routing（地理位置路由）

  • 根據使用者地理位置回傳記錄
  • 先匹配國家，再匹配大洲，最後回傳預設
  • 重疊區域時，優先選擇最精確的位置
  • 適用於授權限制、在地化內容

Geoproximity Routing（地理鄰近路由）

  • 根據使用者和資源的地理位置路由
  • 計算客戶端和記錄之間的距離
  • 可定義規則和偏移（+ 或 - 偏移）
  • 適用於盡可能接近客戶

Active-Active vs Active-Passive Failover

Active-Active Failover

  • 所有資源大部分時間都可用
  • 資源不可用時，R53 偵測到不健康並停止回傳
  • 使用相同名稱、類型、路由策略的記錄
  • R53 可用任何健康記錄回應查詢

Active-Passive Failover

  • 主要資源大部分時間可用
  • 次要資源待命
  • 主要全部不健康時，R53 開始回傳次要
  • 適用於需要主要/備援架構的情況

Route 53 Interoperability

兩種功能

  • Domain Registrar：網域註冊商
  • Domain Hosting：網域託管

架構選項

  1. R53 兩者皆是：R53 註冊 + R53 託管
  2. R53 僅註冊商：R53 註冊，其他服務託管
  3. R53 僅託管：其他註冊商，R53 託管

流程

  • R53 收費（網域註冊費）
  • R53 分配 4 個 Name Servers（網域託管）
  • R53 建立 Zone 檔案（網域託管）
  • R53 與 TLD 登記處通訊（網域註冊商）
  • 設定網域的 NS 記錄指向上述 4 個 NS
