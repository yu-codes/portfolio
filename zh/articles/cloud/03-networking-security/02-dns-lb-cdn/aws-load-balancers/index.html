ELB（Elastic Load Balancing）概述

  • 提供負載平衡器，具有固定 DNS 名稱（如 myalb-123456789.us-east-1.elb.amazonaws.com）
  • 將請求導向 Target Group（可為 EC2、IP、Lambda）

負載平衡器類型

類型支援協定特性
Application Load Balancer (ALB)HTTP, HTTPS, WebSocket適合 Web 應用，支援進階路由
Network Load Balancer (NLB)TCP, UDP, TLS超低延遲，處理大量流量
Gateway Load Balancer第三方代理協定整合第三方安全設備（如防火牆）
Classic Load Balancer (CLB)HTTP, HTTPS, TCP舊式 ELB，不建議新專案使用

Application Load Balancer (ALB)

主要特性

  • 支援基於路徑、主機名、Header 等路由
  • 加入 X-Amzn-Trace-Id 追蹤標頭
  • 來源 IP/Port 可透過 X-Forwarded-For 標頭取得
  • 可與 WAF 結合設限流、防 DDoS

路由規則

條件類型說明
Path基於 URL 路徑路由（如 /api/*）
Host基於主機名路由（如 api.example.com）
HTTP Header基於請求標頭路由
HTTP Method基於請求方法路由（GET/POST）
Query String基於查詢參數路由
Source IP基於來源 IP 路由

目標類型

  • EC2 實例
  • IP 地址
  • Lambda 函數
  • 其他 ALB（Application Load Balancer chaining）

Network Load Balancer (NLB)

主要特性

  • 處理波動大、極低延遲的流量
  • 每個 AZ 可有 Elastic IP，支援固定 IP
  • 可註冊 IP 為目標
  • 可接在 ALB 前做 IP 白名單
  • 支援 TLS 終止

使用場景

  • 需要固定 IP 的應用
  • 極低延遲需求（<1ms）
  • TCP/UDP 協定
  • 高吞吐量應用

ALB vs NLB 比較

項目ALBNLB
OSI 層級Layer 7Layer 4
協定HTTP/HTTPSTCP/UDP/TLS

Stickiness（會話綁定）

  • 僅支援 ALB、CLB
  • 在 Target Group 設定層級設綁定期限
  • 使用 Cookie 實現會話親和性

Cookie 類型

類型說明
Duration-based由 ELB 產生，指定過期時間
Application-based由應用程式產生 Cookie 名稱

Target Group 健康檢查

設定參數

參數說明
Protocol健康檢查協定
Path健康檢查路徑
Interval檢查間隔
Timeout逾時時間
Healthy threshold健康門檻次數
Unhealthy threshold不健康門檻次數

Cross-Zone Load Balancing

  • 啟用時：流量平均分配到所有 AZ 的目標
  • 停用時：流量在 AZ 內分配
類型預設狀態費用
ALB啟用免費
NLB停用跨 AZ 流量需付費
CLB停用免費

Gateway Load Balancer (GLB)

  • 用於部署和管理第三方虛擬設備（防火牆、IDS/IPS）
  • 在 Layer 3（網路層）運作
  • 使用 GENEVE 協定封裝流量
  • 支援透明網路閘道和負載平衡

使用場景

  • 集中化防火牆檢查
  • 入侵偵測和防禦
  • 深度封包檢測
  • 流量鏡像和分析

最佳實踐

  1. Web 應用使用 ALB：利用進階路由功能
  2. 高效能場景使用 NLB：極低延遲需求
  3. 啟用 Cross-Zone Load Balancing：確保流量均勻分配
  4. 設定適當的健康檢查：快速偵測不健康目標
  5. 使用 Target Group：靈活管理後端目標
  6. 安全設備使用 GLB：統一管理第三方網路設備

ELB 架構深入解析

概述

  • ELB 是 AWS 提供的託管負載平衡器
  • 支援多種 Load Balancer 版本
  • 跨 AZ 節點分散
  • 與 ASG、ECS、EKS 等整合

ELB 架構

DNS（A 記錄）→ ELB 節點（每 AZ）→ 目標（EC2、容器等）

節點分佈

功能說明
多 AZ 部署每個 AZ 有 LB 節點
DNS 輪詢DNS 返回所有節點 IP
自動擴展節點根據負載擴展

NLB 架構詳解

Layer 4 負載平衡

特性說明
協定TCP/UDP/TLS
效能數百萬 RPS，超低延遲
靜態 IP每 AZ 一個固定 IP

靜態 IP 架構

客戶端 → 固定 IP（每 AZ）→ NLB 節點 → 目標

TCP 連線

  • NLB 不修改封包
  • 目標看到真實客戶端 IP
  • 支援 Proxy Protocol v2

Connection Draining（連線排空）

概念

  • 也稱為 Deregistration Delay
  • 目標取消註冊時，允許現有連線完成
  • 不接受新連線

設定

設定預設範圍
排空時間300 秒0-3600 秒

建議

  • 長時間請求：增加排空時間
  • 短時間請求：減少排空時間（加快擴縮）

SSL/TLS 終止

ALB SSL

元件說明
SSL 憑證儲存在 ACM 或 IAM
SNI支援多域名憑證
SSL Policy定義支援的協定和加密套件

SNI（Server Name Indication）

  • 允許多個 SSL 憑證在同一 ALB
  • 客戶端在握手時指定主機名
  • ALB 選擇對應憑證

Gateway Load Balancer 深入解析

架構

流量入口 → GLB 端點 → GLB → 安全設備 → GLB → 目的地

GENEVE 協定

  • 封裝原始封包
  • 保留完整封包資訊
  • 支援透明檢查

場景說明
