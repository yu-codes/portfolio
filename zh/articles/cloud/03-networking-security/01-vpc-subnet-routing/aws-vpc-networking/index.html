概述

  • VPC：區域限定虛擬網路，可自訂 CIDR、支援 DNS，預設每區最多建立 5 個
  • 免費元件：VPC、Route Table、NACL、IGW、SG、Subnet、Peering
  • 需付費元件：NAT Gateway、VPC Endpoint、VPN Gateway、Customer Gateway

CIDR（無類別網域間路由）

  • IP 範圍由 Base IP + 子網遮罩組成
  • 如 192.168.0.0/24 表示 256 個 IP
  • 遮罩越小，IP 數越多

核心元件

Subnet（子網）

  • VPC 中的子網段，綁定一個 AZ
  • AWS 每個子網保留 5 個 IP（網路地址、VPC 路由器、DNS、保留、廣播）

Route Table（路由表）

  • 子網的路由規則
  • 可指定至 IGW、NAT、VPC Endpoint 等目標

Internet Gateway (IGW)

  • 讓 Public Subnet 可連上網際網路
  • 一個 VPC 只能有一個 IGW

NAT Gateway / Instance

  • 讓 Private Subnet 中的實例可連網但外部不能連進來
  • NAT Gateway：託管服務，支援 HA，建議使用
  • NAT Instance：EC2 實例，需自行管理

Bastion Host

  • 設於 Public Subnet 的 EC2
  • 透過 SSH 連進 Private Subnet 的機器

Egress-Only Internet Gateway

  • 僅適用 IPv6 的 NAT 功能
  • 讓私有子網中的 IPv6 實例可以送出封包

VPC Peering

  • 兩個 VPC 間私有 IP 通訊
  • 不具轉送性：A-B 和 B-C peering 不代表 A-C 可以通訊
  • 需更新 Route Table
  • 支援跨帳號、跨區域

Transit Gateway

  • Hub-and-Spoke 模式串接多個 VPC
  • 可跨帳號跨區域
  • 支援 IP Multicast
  • 簡化複雜的 VPC 互連架構

VPC Endpoints

可讓 EC2 無需透過 IGW/NAT 即可私下連 AWS 服務。

類型架構支援服務費用
Interface EndpointENI 架構，使用 AWS PrivateLink大多數 AWS 服務付費
Gateway Endpoint路由層級S3、DynamoDB免費

VPC Flow Logs

  • 記錄子網或介面之間的 IP 封包流量
  • 可輸出至 S3 或 CloudWatch
  • 用於安全分析和流量監控

IPv6 支援

  • 皆為 Public IP
  • 不能停用 IPv4
  • 使用 Egress-Only IGW 提供出站連線

AWS VPN

Site-to-Site VPN

  • 本地網路與 VPC 間透過網際網路建立 VPN 隧道
  • 包含 Customer Gateway、Virtual Private Gateway、VPN Connection
  • 每條 VPN 包含 2 條隧道以支援 HA

AWS Direct Connect

  • 提供專線連 AWS，頻寬 1G ～ 100G
  • 可同時存取 AWS 公網與私網資源
  • 建立時間需數日，較適合穩定、高頻寬場景

Amazon CloudFront

  • AWS CDN 服務，將內容快取至邊緣節點（Edge Location）加速回應
  • 支援：S3、ALB、EC2 作為來源，並搭配 OAI 限制來源存取
  • 支援地理限制、簽章 URL / Cookie、TTL 過期與明確 Invalidate
  • 整合 AWS WAF 提供 Layer 7 安全性

AWS Global Accelerator

  • 全球通用加速器服務，透過 AWS 全球骨幹網路選擇最佳路徑
  • 提供兩個 Anycast 靜態 IP
  • 支援註冊多個端點（NLB、ALB、EC2、EIP）與流量加權分配
  • 提供健康檢查與容錯路由

最佳實踐

  1. 規劃 CIDR 範圍：避免與其他網路重疊
  2. 使用 NAT Gateway：比 NAT Instance 更可靠
  3. 啟用 VPC Flow Logs：用於安全分析和疑難排解
  4. 使用多個 AZ：提高可用性
  5. 使用 VPC Endpoints：減少資料傳輸費用和延遲

AWS Transfer Family

AWS Transfer for SFTP/FTPS/FTP

  • 完全託管的檔案傳輸服務
  • 支援 SFTP、FTPS、FTP 協定
  • 資料直接存入 S3 或 EFS
  • 與現有身份認證系統整合（AD、LDAP、自訂）
  • 支援 VPC 端點進行私有存取
  1. 啟用 VPC Flow Logs：監控網路流量
  2. 使用 VPC Endpoints：減少資料傳輸費用
  3. 跨 AZ 部署：提高可用性

VPC 規劃與結構

VPC 規劃考量

  • CIDR 範圍：VPC 最小 /28（16 IPs），最大 /16（65,536 IPs）
  • 網路隔離：考慮 VPC、雲端、本地、合作夥伴和供應商的網路
  • 預留未來擴展空間
  • 層級結構：Web、App、DB、備用
  • 可用區分配：建議至少 3 個 AZ，預留 1 個

  • 避免使用常見範圍，預防未來衝突
  • 每個使用的區域每帳號預留 2+ 個網路

Custom VPC 詳解

基本特性

  • 區域服務：涵蓋該區域所有 AZ
  • 隔離網路：無明確設定不會有任何流量進出
  • 可設定為預設租用或專用租用
  • 支援 IPv4 私有 CIDR 和公有 IP
  • 可選配次要 IPv4 CIDR 區塊
  • 可選配單一 IPv6 /56 CIDR 區塊

VPC 中的 DNS

  • 由 Route 53 提供
  • DNS 位於 VPC Base IP +2 位址
  • enableDnsHostnames：為實例提供 DNS 名稱
  • enableDnsSupport：在 VPC 中啟用 DNS 解析

VPC Subnet 詳解

基本特性

  • AZ 韌性：子網是 VPC 的子網路，位於特定 AZ
  • 1 個子網 → 1 個 AZ，1 個 AZ → 0+ 個子網
  • IPv4 CIDR 是 VPC CIDR 的子集
  • 子網不能重疊
  • 可選 IPv6 CIDR（VPC /56 的 /64 子集）

保留 IP 位址（共 5 個）

以 10.16.16.0/20 為例：

位址用途
10.16.16.0網路位址
10.16.16.1VPC 路由器
10.16.16.2DNS
10.16.16.3保留（未來使用）
10.16.31.255廣播位址

DHCP Option Set

  • 設備如何自動取得 IP 位址
  • 每個子網可設定自動指派公有 IPv4/IPv6

VPC Routing 與 Internet Gateway

VPC Router

  • 每個 VPC 都有 VPC Router，高可用
  • 位於每個子網的 network+1 位址
  • 路由子網間的流量
  • 由 Route Table 控制
  • VPC 有一個 Main Route Table（子網預設使用）
  • /n 數字越大 = 越精確 = 優先度越高
  • 一個子網同時只能關聯一個 Route Table

Internet Gateway（IGW）

  • 區域韌性的閘道，附加到 VPC
  • 1 個 VPC = 0 或 1 個 IGW，1 個 IGW = 0 或 1 個 VPC
  • 運行在 AWS Public Zone
  • 在 VPC 與網際網路/AWS Public Zone 之間傳輸流量
  • AWS 托管，自動處理效能
  • 將私有 IP 對應到公有 IP（反之亦然）

重要概念

  • EC2 的作業系統永遠不知道自己的公有 IPv4
  • 停止和啟動 EC2 = 公有 IP 變更

Bastion Host / Jumpbox

  • 位於公有子網的實例
  • 透過它連入內部 VPC 資源
  • 通常是進入 VPC 的唯一入口

Stateful vs Stateless 防火牆

TCP 基礎

  • TCP 是連線導向協議
  • 使用客戶端隨機埠和伺服器已知埠建立連線
  • 連線建立後是雙向的

HTTP: Port 80 | HTTPS: Port 443

Stateless 防火牆

  • 需要為每個連線設定 2 條規則（1 IN、1 OUT）
  • 入站應用：2 條規則
  • 出站應用：2 條規則（1 OUT、1 IN）

Stateful 防火牆

  • 能識別請求和回應是相關的
  • 自動允許相關的回應流量

Network ACL（NACL）

基本概念

  • 可視為 VPC 中的傳統防火牆
  • 每個子網都有關聯的 NACL
  • 入站規則和出站規則
  • 規則匹配目標 IP/範圍、目標埠和協議
  • 規則按編號順序處理，匹配後停止
  • * 是隱含拒絕（無其他匹配時）

NACL 特性

  • Stateless：請求和回應需要各自的規則
  • 需要設定應用埠和臨時埠的規則對
  • VPC 建立時有預設 NACL（允許所有）
  • 自訂 NACL 預設只有隱含拒絕

NACL 重點

  • Stateless：請求和回應視為不同
  • 只影響跨子網邊界的流量
  • 可明確 ALLOW 和 DENY
  • 只能用 IP/CIDR、埠和協議（無邏輯資源）
  • 無法指派給 AWS 資源，只能指派給子網
  • 與 Security Groups 配合使用可實現明確拒絕
  • 每個子網只能有一個 NACL
  • 一個 NACL 可關聯多個子網

VPC Security Groups（SG）

基本概念

  • 附加到 AWS 資源（實際上是 ENI）
  • 比 NACL 更多優勢：可識別 AWS 資源、可參考其他 SG

SG 特性

  • Stateful：自動偵測回應流量
  • 允許的請求 = 允許的回應
  • 無明確拒絕，只有允許或隱含拒絕
  • 無法阻擋特定惡意來源（需 NACL 協助）
  • 支援 IP/CIDR 和邏輯資源（包括其他 SG 和自身）
  • 附加到 ENI（彈性網路介面），不是實例

邏輯參考

  • 可擴展的權限設定
  • 使用相同 SG 的新實例自動允許通訊
  • 減少管理開銷

SG 自我參考

  • 相同 SG 的資源可以互相通訊

NAT 與 NAT Gateway

NAT 概念

  • IP 偽裝：將 CIDR 區塊隱藏在單一 IP 後
  • 讓私有 VPC 範圍可以出站存取網際網路

NAT Gateway 特性

  • 運行在公有子網
  • 使用彈性 IP（靜態 IPv4 公有）
  • 不支援 Security Groups，只能用 NACL
  • AZ 韌性（該 AZ 內高可用）
  • 區域韌性需要每個 AZ 都有 NATGW
  • 托管服務，可擴展到 45 Gbps
  • 按持續時間和資料量收費

NAT Instance vs NAT Gateway

項目NAT InstanceNAT Gateway
可用性自行管理AWS 管理
頻寬取決於實例類型最高 45 Gbps
維護自行負責AWS 負責
成本較低但需管理較高但免維護
Security Group支援不支援
Bastion Host可兼任不可

IPv6 注意事項

  • NAT 不適用於 IPv6
  • 所有 AWS IPv6 位址都是公有可路由的
  • Internet Gateway 直接處理 IPv6
  • NAT Gateway 不支援 IPv6
  • 使用 Egress-Only Internet Gateway 提供僅出站連線

VPC Flow Logs 詳解

概述

  • 捕獲元資料（非內容）
  • 可附加到 VPC/子網/ENI
  • Flow Logs 非即時
  • 目的地：S3 或 CloudWatch Logs
  • 可用 Athena 查詢

記錄內容

  • 來源/目標 IP、埠、協議、動作等

Egress-Only Internet Gateway

概述

  • IPv4 有私有和公有位址
  • NAT 讓私有 IP 可存取公有網路，但不允許外部主動連入
  • IPv6 所有位址都是公有的
  • Internet Gateway（IPv6）允許所有 IP 進出
  • Egress-Only 是 IPv6 的僅出站閘道

VPC Endpoints 深入解析

Gateway Endpoints

  • 提供對 S3 和 DynamoDB 的私有存取
  • 在 Route Table 加入 Prefix List → Gateway Endpoint
  • 跨所有 AZ 高可用
  • Endpoint Policy 控制可存取的內容
  • 區域性，無法跨區域存取
  • 防止資料外洩：S3 可設定只允許從 Gateway Endpoint 存取

Interface Endpoints

  • 提供對 AWS Public Services 的私有存取
  • 歷史上用於非 S3/DynamoDB 的服務，現在 S3 也支援
  • 加入到特定子網，是一個 ENI，非預設高可用
  • 需在每個使用的 AZ 各加一個 Endpoint
  • 透過 Security Groups 控制網路存取
  • Endpoint Policies 限制操作
  • 只支援 TCP 和 IPv4
  • 使用 PrivateLink
  • 使用 DNS：提供新的服務端點 DNS

VPC Peering

概述

  • 兩個 VPC 之間的直接加密網路連結
  • 支援同/跨區域和同/跨帳號
  • 可選：公有主機名稱解析為私有 IP
  • 同區域 SG 可參考對等 SG
  • 不支援傳遞式對等連線
  • 需要路由設定，SG 和 NACL 可過濾流量
