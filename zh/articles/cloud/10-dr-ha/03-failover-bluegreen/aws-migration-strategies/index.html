概述

AWS 提供多種服務和工具來協助企業將工作負載遷移至雲端。

AWS Snow Family

用於將本地大型資料遷移至 S3，適用於網路頻寬受限的環境。

成員比較

成員儲存空間記憶體遷移方式支援 DataSync資料量等級
Snowcone8TB4GB線上與離線是GB～TB
Snowball Edge Storage Optimized80TB80GB離線否PB
Snowball Edge Compute Optimized42TB208GB離線否PB
Snowmobile100PBN/A離線否EB

注意事項

  • 需安裝 AWS OpsHub 軟體
  • 從本機傳輸至 Snow 裝置
  • 不可直接遷移至 Glacier
  • 需先遷入 S3 並設定生命週期規則轉入 Glacier
  • 如需直接傳輸至 Glacier，請使用 DataSync

AWS Storage Gateway

混合雲服務，連接本地端應用與雲端儲存資源。

Gateway 類型協定後端儲存用途說明
File GatewayNFS & SMBS3 → S3-IA / One Zone-IA將本地檔案儲存為 S3 物件，支援快取與 AD 驗證
FSx File GatewaySMB & NTFSFSx → S3整合 Microsoft AD 的 Windows/Lustre 檔案伺服器
Volume GatewayiSCSIS3 → EBS將本地磁碟區備份至 S3
Tape GatewayiSCSI VTLS3 → Glacier模擬磁帶備份

Volume Gateway 模式

模式說明
Cached Volume資料存於 S3，常用資料快取本地
Stored Volume資料存於本地，異步備份至 S3

AWS DataSync

功能

  • 用於大規模資料遷移
  • 從本地（NFS、SMB）至 AWS（S3、EFS、FSx）
  • 可直接將冷資料歸檔至 Glacier 或 Glacier Deep Archive
  • 支援傳輸至任意 S3 儲存類別

與 Direct Connect 整合

  • 搭配 Direct Connect 可透過私網路傳輸
  • 安全傳輸至 VPC 中的 AWS 儲存服務

DataSync vs Storage Gateway

項目DataSyncStorage Gateway
用途一次性或週期性遷移持續混合雲整合
架構遷移完成後可移除長期運行
目標S3、EFS、FSxS3、FSx、EBS

AWS Database Migration Service（DMS）

功能

  • 支援不中斷運作下的資料庫遷移
  • 來源在遷移過程仍可使用
  • 需在 EC2 上執行 DMS 以進行複製

遷移類型

類型說明範例
同構遷移相同引擎PostgreSQL → PostgreSQL
異構遷移不同引擎Oracle → Aurora

常見遷移路徑

  • SQL Server → MySQL
  • Oracle → Aurora
  • Teradata → Redshift
  • MongoDB → DynamoDB
  • S3 → DynamoDB

AWS Schema Conversion Tool（SCT）

  • 進行異構遷移需先使用
  • 在來源端分析並轉換結構
  • 自動轉換 Schema

AWS Application Migration Service（MGN）

功能

  • 遷移虛擬機器至 AWS
  • 支援 VMware vSphere、Hyper-V、Azure

新版 MGN

  • 使用連續區塊層級複製
  • 可在分鐘內完成切換
  • 推薦使用

舊版 Server Migration Service（SMS）

  • 使用快照為主
  • 切換時間為數小時
  • 已不建議新案使用

遷移策略（6R）

策略說明複雜度
Rehost直接遷移（Lift & Shift）低
Replatform小幅調整後遷移低-中
Repurchase改用 SaaS低
Refactor重新架構為雲原生高
Retire淘汰不需要的系統-
Retain保留在本地-

遷移流程

1. 評估階段

  • 盤點現有資源
  • 評估相依性
  • 規劃遷移順序

2. 遷移階段

  • 執行遷移
  • 驗證資料完整性
  • 測試功能

3. 優化階段

  • 調整資源大小
  • 實施成本優化
  • 建立監控

最佳實踐

  1. 評估網路頻寬：決定使用線上或離線遷移
  2. 使用 DataSync：大規模資料遷移
  3. 使用 DMS：資料庫遷移
  4. 使用 MGN：VM 遷移
  5. 規劃切換時間：最小化停機時間
  6. 驗證資料：確保遷移完整性
  7. 保留回滾計畫：應對遷移問題
