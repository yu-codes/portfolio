概述

  • AWS 託管的關聯式資料庫服務
  • 支援 PostgreSQL、MySQL、MariaDB、Oracle、SQL Server、Amazon Aurora

可擴展性

Read Replica

  • 支援最多 5 個 Read Replica
  • 非同步複製，讀取為最終一致性
  • 可部署於不同區域
  • 跨區複製需額外付費，但跨 AZ 不需額外費用

高可用性（Multi-AZ）

  • 使用 Multi-AZ 部署，支援同步複製
  • 發生異常時會自動將 CNAME 指向備援 DB 實例
  • 達成自動故障轉移

備份與還原

自動備份

  • 每天備份資料庫儲存快照
  • 保留期為 1～35 天
    • Console 預設 7 天
    • CLI 預設 1 天
  • 若需 90 天可搭配 AWS Backup

加密未加密的資料庫

  1. 建立 Snapshot
  2. 複製並加密此快照
  3. 從加密快照還原為新資料庫

安全性

IAM Database Authentication

  • 讓 IAM 使用者透過 IAM 權限登入資料庫
  • 僅支援 MySQL 與 PostgreSQL

監控

Enhanced Monitoring

  • 檢視每個資料庫程序的 CPU 與記憶體使用率
  • 提供更詳細的系統層級指標

Amazon Aurora

概述

  • AWS 自有、相容 MySQL 與 PostgreSQL 的高效能關聯式資料庫
  • 效能為 MySQL 的 5 倍，PostgreSQL 的 3 倍

Aurora 架構

一般為一組 Cluster 架構：

元件說明
Primary DB instance主節點，支援讀寫
Aurora Replica最多 15 個，只支援讀取，自動故障轉移

Aurora Endpoint

Endpoint 類型說明
Cluster Endpoint連至主節點，可執行讀寫操作
Reader Endpoint讀取節點入口，支援讀取負載平衡
Custom Endpoint最多 5 個，自訂節點群組
Instance Endpoint直接連至指定實例

Aurora Global Database

  • 跨區域部署一個資料庫
  • 實現低延遲全球讀取與災難復原
  • RPO 1 秒，RTO 1 分鐘

Aurora Serverless

  • 可自動擴展資源
  • 適用於間歇性或難以預測的負載
  • Serverless v2：更快的擴展、更精細的容量調整

RDS 進階功能

Multi-AZ 部署選項

類型說明
Multi-AZ with Standby單一備援實例，同步複製
Multi-AZ DB Cluster兩個備援實例，寫入確認後才回傳

支援的資料庫引擎

  • MySQL、PostgreSQL、MariaDB
  • Oracle Database、Microsoft SQL Server
  • Amazon Aurora
  • SAP HANA（RDS Custom）

AWS Secrets Manager 整合

  • 自動輪替資料庫認證
  • 安全儲存資料庫連線資訊
  • 與 RDS 原生整合

RDS vs Aurora 比較

項目RDSAurora
引擎多種（MySQL、PostgreSQL、Oracle 等）MySQL 或 PostgreSQL 相容
效能標準MySQL 5x / PostgreSQL 3x
儲存手動擴展自動擴展（最高 128TB）
Replica最多 5 個最多 15 個
故障轉移較慢較快（通常 <30 秒）
成本較低較高

最佳實踐

  1. 使用 Multi-AZ：確保高可用性
  2. 配置 Read Replica：分散讀取負載
  3. 啟用自動備份：保護資料
  4. 使用 IAM Authentication：簡化憑證管理
  5. 監控效能：使用 Enhanced Monitoring
  6. 考慮 Aurora：需要更高效能時
  7. 使用 Secrets Manager：自動輪替資料庫認證
  8. 考慮 Aurora Serverless v2：變動負載場景

RDS 架構深入解析

資料庫即服務（DBaaS）

  • RDS 提供資料庫伺服器即服務
  • 每個 RDS 實例可以包含多個使用者建立的資料庫
  • 使用標準資料庫引擎（MySQL、MariaDB、PostgreSQL、Oracle、Microsoft SQL Server）

RDS 如何運作

  • 無法存取作業系統或 SSH
  • RDS 在子網路中運行，使用 DB Subnet Group
  • RDS 實例放置在 VPC 內

成本

  • 按使用時數計費
  • 成本因素：引擎、類型、大小、Multi-AZ
  • 預設使用 EBS：gp2（1GB - 16TB）

Multi-AZ 詳解

架構

  • 在同一區域內的不同 AZ 中建立主要和備用實例
  • 資料庫 CNAME 指向主要實例
  • 同步複製到備用實例
  • 發生故障轉移時，RDS 自動將 DNS 變更為備用實例

重要特性

  • 不能使用備用實例進行讀取（在故障轉移之前）
  • 60-120 秒的故障轉移時間
  • 同一區域兩個 AZ 內的複本
  • 備份從備用實例進行（減少主要實例負擔）

Multi-AZ Cluster 模式

  • 1 個 Writer（可讀寫）+ 2 個 Reader 實例
  • 跨 3 個 AZ 運作
  • 比傳統實例模式更快速的硬體
  • 寫入被認為已提交後返回
  • Reader 可用於讀取（擴展讀取）
  • 更快的故障轉移時間（約 35 秒 + 交易日誌應用）

RDS 備份與還原詳解

兩種備份方式

  1. 自動備份
  2. 快照（手動）

RDS 備份特性

  • 備份到 S3
  • 自動備份每 5 分鐘進行交易日誌備份
  • 刪除實例時，自動備份可選擇保留
  • 備份來自備用實例（若啟用 Multi-AZ）

還原

  • 建立新的 RDS 實例
  • 新的端點位址
  • 快照還原：單一時間點
  • 自動備份還原：任何 5 分鐘的時間點

Read Replica 深入解析

效能改善

  • Read Replica 是唯一提供效能擴展（而非可靠性）的方式
  • 最多 5 個 Direct Read Replica（每個 Read Replica 可再有 5 個）
  • 可與 Multi-AZ 結合使用

跨區域 Read Replica

  • 完全不同的區域
  • 資料保留在特定區域，不會離開
  • AWS 處理所有從源到副本的網路
  • 需要考慮跨區域流量成本

同步類型比較

類型同步方式用途
Multi-AZ同步高可用性
Read Replica非同步效能擴展

重要考量

  • Read Replica 只能讀取
  • Read Replica 可以有自己的 Read Replica
  • 全球效能改善
  • 讀取延遲：因為是非同步複製

RDS 資料安全

傳輸中加密

  • SSL/TLS 可用於 RDS
  • 可配置為強制 SSL

靜態加密

  • 由 EBS 和 KMS 處理
  • KMS 與 RDS 主機一起處理加密
  • 資料離開主機後即被加密
  • 包括：儲存、日誌、快照、副本

IAM 認證

  • 預設使用本地使用者帳號（使用者名稱和密碼）
  • IAM 認證：可配置本地 RDS 使用者使用 AWS 認證權杖
  • 授權仍由資料庫引擎處理

RDS Custom

概述

  • 填補 RDS 和在 EC2 上自行安裝 DB 之間的差距
  • 目前支援 MS SQL 和 Oracle
  • 可以連接使用 SSH、RDP、Session Manager

使用案例

  • 需要存取作業系統或資料庫引擎
  • 複雜的自訂需求

Aurora 架構詳解

與 RDS 的主要差異

  • Aurora 使用叢集架構
  • 單一主要實例 + 0 個或更多副本
  • 無本地儲存：使用叢集磁碟區
  • 儲存更快、更有彈性

叢集磁碟區

  • 最大 128 TB
  • 6 個副本，跨 3 個 AZ
  • 最多 15 個副本
  • 自動擴展，無需手動調整
  • 高水位計費：只收取曾使用的最高空間

端點類型

端點說明
Cluster Endpoint指向主要實例（讀寫）
Reader Endpoint指向副本（負載平衡）
Custom Endpoint自訂實例群組
Instance Endpoint特定實例

成本

  • 無免費層
  • 計算按秒計費（最少 10 分鐘）
  • 儲存：每 GB-月計費
  • IO 成本：每請求計費

Aurora Serverless

概念

  • 可擴展的 ACU（Aurora Capacity Units）
  • 每個 ACU = 約 2 GiB 記憶體
  • 由 AWS 管理的暖機池

架構

  • 應用程式連接到 Proxy Fleet（代理艦隊）
  • Proxy Fleet 路由到 ACU
  • ACU 根據負載自動增減
  • 不使用時可擴展到 0

使用案例

  • 不可預測的工作負載
  • 新應用
  • 變動工作負載
  • 開發和測試
  • 多租戶應用

Aurora Global Database

功能

  • 跨區域複製
  • 主要區域：1 個讀寫叢集
  • 次要區域：最多 16 個只讀副本

效能

  • 約 1 秒複製到所有區域
  • RPO：1 秒
  • RTO：< 1 分鐘

Aurora Multi-Master

概念

  • 預設模式：1 個讀寫 + 0-15 個只讀副本
  • Multi-Master：所有實例都可讀寫
  • 無叢集端點可用

運作方式

  • 應用連接到多個端點
  • 每個節點可讀寫
  • 變更傳送到其他節點
  • 如果寫入衝突，會向應用回報錯誤

使用案例

  • 需要故障容忍能力
  • 在節點故障期間無中斷

Database Migration Service (DMS)

概述

  • 受管理的資料庫遷移服務
  • 在複製實例中運行一或多個複製任務
  • 定義源和目標端點

遷移類型

類型說明
Full Load一次性全量遷移
Full Load + CDC全量遷移 + 持續資料更改捕獲
CDC only僅持續複製變更

Schema Conversion Tool (SCT)

  • 用於異質資料庫遷移
  • 轉換 Schema 結構
  • 例如：Oracle 到 Aurora MySQL

同質 vs 異質遷移

遷移類型說明需要 SCT
同質相同引擎間（如 MySQL → MySQL）否
異質不同引擎間（如 Oracle → Aurora）是

DMS + Snowball

  • 用於大規模資料遷移
  • 透過 Snowball 傳輸大量資料
  • 結合 CDC 處理遷移期間的變更
