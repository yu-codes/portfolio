概述

  • S3 是物件型儲存服務，每筆資料為一個 Object
  • Bucket 名稱需在全球唯一（類似網域）
  • 支援無限儲存數量、單一檔案最大 5TB
  • 建議 100MB 以上使用 Multi-part Upload
  • 預設為私有存取，需明確開啟 Public Access

S3 Object 組成

元件說明
Value實際資料（圖像、影片、文件等）
Key完整路徑，如 /movies/comedy/abc.avi
Version ID物件版本（啟用版本控制時）
Metadata額外屬性資訊

存取控制

方式

方式說明
Access Control List（ACL）已淘汰
S3 Bucket Policy推薦使用，JSON 格式政策
IAM Policy控制 IAM 使用者存取

資料保護

  • Versioning：版本控制
  • MFA Delete：刪除時需要 MFA 驗證
  • Object Lock：不可變資料保存（WORM 模式）
    • Retention Mode：保留期限
    • Legal Hold：法律保留

S3 儲存類別

類別耐久性可用性AZ 數最小儲存存取時間存取費用
Standard11 9’s99.99%≥3無毫秒無
Intelligent Tiering11 9’s99.9%≥330 天毫秒無

Glacier Deep Archive11 9’s99.99%≥3180 天最多 48 小時每 GB

Lifecycle Policy（生命週期政策）

轉換規則

  • S3 Standard → IA/One Zone-IA：至少儲存 30 天
  • S3 Standard → Intelligent Tiering / Glacier：可立即轉換

過期規則

  • 自動刪除過期版本
  • 移除未完成 Multipart Upload

加密

傳輸中加密

  • 透過 HTTPS/SSL 加密

靜態加密（Server-side Encryption, SSE）

類型說明
SSE-S3AWS 管理金鑰，AES-256 加密
SSE-KMS使用 KMS 金鑰（Envelope encryption）
SSE-C客戶自管金鑰，需透過 HTTPS 傳輸

用戶端加密

  • 由客戶在本地加密/解密

合規建議

  • PCI-DSS / HIPAA：使用 SSE-C 或用戶端加密

資料一致性

  • 提供強一致性：PUT/DELETE 完成後馬上可見
  • 同一 Key 的更新為原子性操作

S3 Express One Zone（新）

  • 單一 AZ 儲存類別，延遲低於 10 毫秒
  • 適用於機器學習、即時分析等高效能場景
  • 比 S3 Standard 便宜但僅單 AZ
  • 使用 Directory Bucket 而非一般 Bucket

進階功能

靜態網站託管

  • 部署 HTML、CSS、JS
  • 需啟用 Static Hosting + Public Access + CORS

Pre-signed URL

  • 提供限時上傳/下載存取權限

S3 Select / Glacier Select

  • 對 CSV / JSON / Parquet 資料用 SQL 查詢部分內容

Byte Range Fetch

  • GET Request 加 Range Header 只下載檔案部分內容

事件通知

  • 例如 s3:ObjectCreated:* 可觸發 SNS/SQS/Lambda

Cross-Region Replication（CRR）

  • 跨區異步複製物件
  • 來源與目的地都需開啟版本控制

Server Access Logging

  • 紀錄物件存取紀錄（CloudTrail 不支援物件層級）

VPC S3 Gateway Endpoint

  • 提供私網存取，節省資料傳輸費

Transfer Acceleration

  • 加速全球上傳速度

CloudFront + OAI

  • 透過 CDN 快取公開檔案
  • 限制非 CDN 來源存取

S3 Access Grants（新）

  • 簡化對 S3 資料的存取權限管理
  • 支援與企業目錄整合（如 Active Directory）
  • 可按使用者或群組授予存取權限

S3 Object Lambda

  • 在 S3 GET 請求時自動轉換資料
  • 使用 Lambda 函數處理物件內容
  • 適用於資料遮罩、格式轉換等場景

AWS Athena

  • Serverless SQL 查詢引擎
  • 可直接對 S3 上的物件進行查詢分析
  • 結果可輸出至 S3
  • 適用於一次性查詢、存取日誌、IoT 分析等

最佳實踐

  1. 使用適當的儲存類別：根據存取頻率選擇
  2. 配置 Lifecycle Policy：自動轉換和清理
  3. 啟用版本控制：保護重要資料
  4. 使用 VPC Endpoint：節省資料傳輸費
  5. 實施加密：保護敏感資料
  6. 考慮 S3 Express One Zone：高效能低延遲場景
  7. 使用 S3 Object Lambda：即時資料轉換需求

S3 基礎架構

Bucket 命名規則

  • Bucket 名稱全球唯一
  • 3-63 字元，全部小寫，不可有底線
  • 必須以小寫字母或數字開頭
  • 不可使用 IP 格式（如 1.1.1.1）
  • 每帳號 100 個軟限制，1000 個硬限制
  • 單一 Bucket 可存無限物件，單檔 0 bytes 到 5TB

S3 的適用與不適用場景

適用場景不適用場景
大規模資料儲存、分發、上傳檔案系統用途
計算服務的輸入/輸出需要掛載為磁碟（K:\ 或 /images）
「Offload」靜態資源區塊儲存需求

S3 是物件儲存，不是檔案或區塊儲存，無法掛載

S3 安全性深入解析

S3 預設為私有

所有新建的 Bucket 預設為私有，需明確授權才能存取。

Bucket Policy（桶政策）

  • 是一種 Resource Policy
  • 附加到 Bucket 上，從資源角度定義權限
  • 可允許/拒絕相同帳號或不同帳號
  • 可允許/拒絕匿名主體（Anonymous Principals）

Access Control Lists（ACL）

  • 可設定在物件或 Bucket 上
  • 已過時（Legacy），不建議使用
  • 權限控制不夠靈活

Block Public Access

  • 安全防護機制，防止意外公開

何時使用何種方式

場景建議方式
控制多種不同資源Identity Policy
偏好使用 IAMIdentity Policy
同帳號存取Identity Policy
只控制 S3Bucket Policy
匿名或跨帳號Bucket Policy
舊系統必須用 ACLACL（否則永遠不要用）

靜態網站託管深入解析

功能特性

  • 一般存取透過 AWS APIs
  • 此功能允許透過 HTTP 存取
  • 設定 Index 和 Error 文件
  • 建立 Website Endpoint
  • 透過 Route 53 設定自訂網域（Bucket 名稱需與網域相符）

常見用途

  • Offloading：將大型靜態檔案（如圖片）放在 S3 減輕伺服器負擔
  • Out-of-band 頁面：維護期間，DNS 指向 S3 上的錯誤頁面

S3 定價

  • ❗ 按 GB/月 收費
  • ❗ 上傳（IN）免費
  • ❗ 下載（OUT）收費
  • ❗ GET、PUT、POST 等操作按每 1000 次請求收費

Object Versioning 與 MFA Delete

版本控制

  • ⚠️ 一旦啟用，就無法停用！ 只能暫停和重新啟用
  • 未啟用版本控制時，物件 ID 為 null
  • 請求物件時不指定版本，會取得最新版本
  • 刪除物件時不指定 ID，會新增 Delete Marker（可刪除）
  • 完全刪除需指定物件版本 ID
  • 所有版本都會佔用空間並計費

MFA Delete

  • 在版本控制設定中啟用
  • 變更 Bucket 版本控制狀態需要 MFA
  • 刪除版本需要 MFA
  • 需在 API 呼叫中傳入序號和驗證碼

S3 效能最佳化

Single PUT Upload

  • 單一資料流上傳
  • 串流失敗即上傳失敗
  • 適用於最大 5GB 的上傳

Multipart Upload

  • 資料分割上傳
  • 最小 100MB 才能使用
  • 最多 10,000 個部分，每部分 5MB 到 5GB
  • 部分失敗可重傳
  • 傳輸速率 = 所有部分速度總和

S3 Transfer Acceleration

  • 使用 Edge Location 網路
  • 預設關閉，需啟用
  • 透過 AWS 網路傳輸，比公共網路更有效率
  • 更低且一致的延遲
  • 連線品質越差，加速效果越明顯

S3 加密詳解

加密範圍

Bucket 不加密，物件才加密

Client-Side Encryption

  • 客戶端上傳前自行加密
  • 金鑰、流程、工具由客戶管理

Server-Side Encryption

物件以明文到達 S3，然後由 S3 加密

類型金鑰管理加密處理特點
SSE-C客戶S3上傳時需提供金鑰
SSE-S3S3S3AWS 管理，AES-256
SSE-KMSS3 + KMSS3可控制金鑰權限、旋轉，支援角色分離

SSE-KMS 優勢

  • 使用 KMS 金鑰產生每個物件的唯一加密金鑰
  • 可使用客戶管理的 KMS 金鑰
  • 角色分離：S3 管理員無法看到未加密的物件（需要 KMS 金鑰權限）

Bucket Default Encryption

  • PUT 操作時透過 header 指定：
    • x-amz-server-side-encryption: "AES256" → SSE-S3
    • x-amz-server-side-encryption: "aws:kms" → SSE-KMS
  • 可設定 Bucket 預設加密方式
  • 可限制 Bucket 允許的加密類型

S3 儲存類別詳解

S3 Standard

  • 11 個 9 的耐久性
  • 資料複製到至少 3 個 AZ
  • 適用於經常存取的資料

S3 Standard-IA（Infrequent Access）

  • 比 Standard 便宜
  • 有取用費用，頻繁存取成本會增加
  • 最小計費 30 天，最小物件 128KB

S3 One Zone-IA

  • 只存在單一 AZ
  • 比 Standard-IA 便宜 20%
  • 適用於可重建的資料

S3 Glacier Instant

  • 類似 Standard-IA，但儲存更便宜、取用更貴
  • 最小計費 90 天

S3 Glacier Flexible

  • 冷資料（Cold Objects）
  • 物件無法公開存取，需經過取用流程
  • 取用選項：Expedited（1-5分鐘）、Standard（3-5小時）、Bulk（5-12小時）
  • 最小計費 90 天

S3 Glacier Deep Archive

  • 最便宜的選項
  • 取用時間：數小時到數天
  • 最小計費 180 天

S3 Intelligent-Tiering

  • 自動在不同層級間移動物件
  • 有監控和自動化費用
  • 適用於存取模式不明確的資料

S3 Lifecycle Configuration

概述

  • 自動轉換或刪除 Bucket 中的物件
  • 由**規則（Rules）**組成
  • 規則包含動作（Actions）
  • 可套用到整個 Bucket 或特定物件群組

動作類型

  • Transition Actions：轉換到其他儲存類別
  • Expiration Actions：在指定時間後刪除物件

轉換限制

儲存類別轉換有「瀑布式」限制：

Standard → Standard-IA → Intelligent-Tiering → One Zone-IA → Glacier Instant → Glacier Flexible → Glacier Deep Archive

S3 Replication

類型

  • CRR（Cross-Region Replication）：跨區域複製
  • SRR（Same-Region Replication）：同區域複製

跨帳號複製

  • 角色預設不被目標帳號信任
  • 需在目標 Bucket 加入 Bucket Policy 允許角色存取

複製選項

  • 複製全部物件或子集
  • 儲存類別：預設維持原類別，可覆寫
  • 擁有權：預設為來源帳號，可覆寫為目標帳號
  • RTC（Replication Time Control）：確保 15 分鐘內同步

重要考量

  • 不追溯：啟用前的物件不會複製，需開啟版本控制
  • 單向複製：來源到目標，目標新增物件不會回傳
  • 支援 SSE-S3、SSE-KMS（需額外設定），不支援 SSE-C
  • 來源 Bucket 擁有者需有物件權限
  • 不複製：系統事件、Glacier/Deep Archive 物件
  • Delete Marker 預設不複製

使用場景

類型場景
SRR日誌彙整、生產/測試同步、嚴格主權要求的韌性
CRR全球韌性改善、延遲降低

S3 Presigned URLs

概述

  • 安全地使用您的憑證授予他人存取 Bucket 中物件
  • 有過期時間
  • 使用 URL 的人以建立者身份執行 PUT/GET

使用場景

  • 將媒體 Offload 到 S3
  • 授予暫時存取權限

重要注意事項

  • 可以為您沒有權限的物件建立 URL（少數情況會用到）
  • 權限以建立者當下權限為準
  • 存取被拒可能是：建立者從未有權限或現在沒有權限
  • ⚠️ 不要用 Role 建立：臨時憑證過期後 URL 會失效

S3 Select 與 Glacier Select

概述

  • 取回物件的部分內容而非整個物件
  • 使用類 SQL 語句查詢
  • 支援 CSV、JSON、Parquet、BZIP2 壓縮

優勢

  • S3 預先過濾資料再回傳
  • 減少資料傳輸量和成本

S3 Events

功能

  • 當 Bucket 中發生事件時產生通知
  • 可傳送到 SNS、SQS、Lambda

事件類型

  • Object Created（Put, Post, Copy, CompleteMultiPartUpload）
  • Object Delete
  • Object Restore
  • Replication
