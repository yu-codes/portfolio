概述

AWS 提供多種工具和策略來優化雲端成本，實現最佳的成本效益。

定價模式

EC2 定價

類型折扣承諾適用場景
On-Demand無無彈性、短期
Reserved最高 72%1-3 年穩定工作負載
Spot最高 90%無（可中斷）容錯應用
Savings Plans最高 72%1-3 年彈性承諾

Savings Plans

類型適用範圍
Compute Savings PlansEC2、Fargate、Lambda
EC2 Instance Savings Plans特定實例系列

成本管理工具

AWS Cost Explorer

  • 視覺化成本和使用量
  • 預測未來成本
  • 識別成本異常

AWS Budgets

  • 設定預算警示
  • 追蹤成本和使用量
  • 自動化成本控制

Cost Allocation Tags

  • 使用標籤分類成本
  • 按專案、部門、環境追蹤
  • 支援自訂和 AWS 產生的標籤

成本優化策略

1. 右調整（Right Sizing）

  • 選擇適當的實例大小
  • 使用 CloudWatch 監控使用率
  • 定期檢視並調整

2. 使用 Reserved Instance

  • 穩定工作負載
  • 1 年或 3 年承諾
  • 可轉售或轉換

3. 使用 Spot Instance

  • 容錯應用
  • 批次處理
  • CI/CD 環境

4. 自動擴縮

  • 根據需求調整資源
  • 避免過度配置
  • 使用 Target Tracking

儲存成本優化

S3 儲存類別

從到策略
StandardStandard-IA不常存取的資料
Standard-IAGlacier歸檔資料
任意Intelligent-Tiering不確定存取模式

生命週期政策

  • 自動轉換儲存類別
  • 自動刪除過期物件
  • 清理未完成的分段上傳

EBS 優化

  • 刪除未使用的 Volume
  • 使用適當的 Volume 類型
  • 定期清理 Snapshot

資料傳輸成本

免費

  • 同一 AZ 內傳輸
  • 從網際網路傳入

收費

  • 跨 AZ 傳輸
  • 跨區域傳輸
  • 傳出至網際網路

優化策略

  • 使用 VPC Endpoint
  • 同 AZ 部署相關資源
  • 使用 CloudFront 減少出站

資料庫成本優化

RDS

  • 使用 Reserved Instance
  • 適當的實例大小
  • 考慮 Aurora Serverless

DynamoDB

  • 使用 On-demand 或 Provisioned
  • 配置適當的容量
  • 使用 DAX 減少讀取成本

AWS Organizations 成本管理

合併帳單

  • 所有帳號合併計費
  • 獲得批量折扣
  • 共享 Reserved Instance

成本分配

  • 按 OU 分配成本
  • 使用標籤追蹤
  • 設定帳號層級預算

成本優化工具

AWS Compute Optimizer

  • 分析使用模式
  • 提供 EC2、EBS、Lambda 最佳化建議
  • 基於機器學習的建議

AWS Trusted Advisor

  • 成本優化建議
  • 效能、安全、容錯建議
  • 服務限制監控

成本優化最佳實踐

  1. 監控成本：使用 Cost Explorer 定期檢視
  2. 設定預算：使用 AWS Budgets 設定警示
  3. 使用標籤：追蹤和分配成本
  4. 右調整：定期檢視資源使用率
  5. 使用承諾折扣：RI 或 Savings Plans
  6. 清理未使用資源：定期審核和刪除
  7. 優化資料傳輸：減少跨區域傳輸
  8. 使用 Compute Optimizer：獲取最佳化建議
  9. 考慮無伺服器架構：按使用量付費
  10. 使用成本效益的儲存層級：S3 Glacier、EBS gp3
