標籤策略（Tagging Strategy）

概述

標籤是管理 AWS 資源的關鍵，可用於成本分配、存取控制、自動化等。

標籤類型

類型說明
AWS 產生標籤AWS 自動建立（如 aws:createdBy）
使用者定義標籤自訂標籤

常見標籤鍵

標籤鍵用途
Environment環境（dev/staging/prod）
Project專案名稱
Owner資源擁有者
CostCenter成本中心
Application應用程式名稱

成本分配標籤

啟用步驟

  1. 在 Billing Console 啟用標籤
  2. 等待 24 小時生效
  3. 在 Cost Explorer 中使用

最佳實踐

  • 定義統一的標籤標準
  • 使用 Tag Policy 強制執行
  • 自動化標籤套用

AWS Organizations 標籤政策

Tag Policy

  • 強制執行標籤標準
  • 定義允許的標籤鍵和值
  • 套用於 OU 或帳號

範例

{
  "tags": {
    "Environment": {
      "tag_key": {
        "@@assign": "Environment"
      },
      "tag_value": {
        "@@assign": ["dev", "staging", "prod"]
      }
    }
  }
}

帳單管理

帳單儀表板

  • 查看當月費用
  • 預測月底費用
  • 按服務細分

帳單詳細報告

報告類型說明
AWS Cost and Usage Report最詳細的帳單資料
Monthly Report每月摘要
Cost Allocation Report按標籤分配

Cost and Usage Report（CUR）

  • 最詳細的成本資料
  • 輸出至 S3
  • 可整合 Athena、QuickSight

預算管理

AWS Budgets 類型

類型說明
成本預算追蹤支出
使用量預算追蹤服務使用量
預留預算追蹤 RI 使用率
Savings Plans 預算追蹤 SP 使用率

預算動作

  • 發送通知
  • 套用 IAM 政策
  • 套用 SCP

成本異常偵測

Cost Anomaly Detection

  • 機器學習偵測異常
  • 自動基準線
  • 即時警示

設定

  1. 選擇監控範圍（帳號、服務、標籤）
  2. 設定警示閾值
  3. 配置通知目標

AWS Glue 資料目錄

與成本分析整合

  • 掃描 CUR 資料
  • 建立資料目錄
  • 使用 Athena 查詢

多帳號成本管理

Organizations 合併帳單

  • 單一付款帳號
  • 批量折扣
  • RI 共享

成本分配方法

方法說明
標籤按專案/部門分配
帳號按帳號分配
OU按組織單位分配

最佳實踐

標籤

  1. 定義標籤標準：統一命名規則
  2. 使用 Tag Policy：強制執行標準
  3. 自動化標籤：減少人為錯誤
  4. 定期審核：確保標籤完整性

帳單

  1. 啟用 CUR：詳細成本分析
  2. 設定預算警示：預防超支
  3. 使用 Cost Explorer：視覺化分析
  4. 定期檢視：識別優化機會
