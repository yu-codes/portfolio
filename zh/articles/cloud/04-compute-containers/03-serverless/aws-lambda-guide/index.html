概述

  • FaaS（無伺服器函數服務）
  • 支援 Node.js、Python、Java、C#、Go、Ruby 等語言

限制

項目限制
最大執行時間15 分鐘
記憶體128MB～10GB（1MB 為單位）
/tmp 目錄大小512MB
環境變數最多 4KB
Zip 壓縮上限50MB
解壓後大小250MB
並發執行1000（可申請調升）

Lambda 觸發來源

來源說明
DynamoDB資料變更觸發
S3物件事件觸發
EventBridge排程或事件觸發
SNS訊息通知觸發
SQS佇列訊息觸發
API GatewayHTTP 請求觸發
ALBHTTP 請求觸發
Kinesis資料串流觸發
CloudWatch Logs日誌觸發

Lambda 優勢

  1. 秒級擴展：不需像 EC2 等待幾分鐘
  2. 按量計費：僅根據請求次數與執行時間計費，成本低
  3. 無需管理伺服器：專注於程式碼
  4. 高可用性：內建跨 AZ 容錯

Lambda@Edge

  • 在 CloudFront 邊緣執行 Lambda 函數
  • 用於自訂 CDN 行為

使用場景

觸發點說明
Viewer Request使用者請求到達 CloudFront
Origin RequestCloudFront 請求到達 Origin
Origin ResponseOrigin 回應到達 CloudFront
Viewer ResponseCloudFront 回應到達使用者

錯誤處理

Dead Letter Queue（DLQ）

  • 捕捉處理失敗請求
  • 支援 SQS 或 SNS 作為 DLQ

重試行為

觸發類型重試行為
同步呼叫不重試
非同步呼叫重試 2 次
串流（Kinesis/DynamoDB）直到資料過期

Lambda 與 VPC

  • Lambda 可部署在 VPC 中存取私有資源
  • 需要 ENI（Elastic Network Interface）
  • 需要足夠的 IP 地址

VPC 配置

  • 選擇子網
  • 配置安全群組
  • 確保有 NAT Gateway 存取網際網路（如需要）

Lambda Layers

  • 可重用的程式碼和相依性
  • 最多 5 個 Layer
  • 減少部署套件大小

版本與別名

版本（Version）

  • 不可變的函數快照
  • 包含程式碼和配置

別名（Alias）

  • 指向特定版本的指標
  • 支援加權流量分配（金絲雀部署）

支援的執行時期（2024-2025 更新）

執行時期版本
Node.js22.x, 20.x, 18.x
Python3.13, 3.12, 3.11, 3.10, 3.9
Java21, 17, 11
.NET8, 6
Ruby3.3, 3.2
Go1.x（使用 provided.al2023）
Rust使用 provided.al2023

ARM64 (Graviton2) 支援

  • 支援 arm64 架構，提供更好的性價比
  • 比 x86_64 便宜約 20%
  • 大多數執行時期都支援

容器映像支援

  • 支援最大 10GB 的容器映像
  • 可使用自訂執行時期
  • 需要從 ECR 提取映像

監控與日誌

  • 執行紀錄自動寫入 CloudWatch Logs
  • 需要 IAM Role 具備以下權限：
    • logs:CreateLogGroup
    • logs:CreateLogStream
    • logs:PutLogEvents

CloudWatch 指標

指標說明
Invocations呼叫次數
Duration執行時間
Errors錯誤次數
Throttles節流次數
ConcurrentExecutions並發執行數

IAM Role

  • 配 IAM Role 以存取 EC2、S3 等資源
  • 執行角色（Execution Role）：Lambda 執行時使用
  • 資源政策：控制誰可以呼叫 Lambda

最佳實踐

  1. 設定適當記憶體：記憶體也影響 CPU 分配
  2. 使用環境變數：配置敏感資訊
  3. 實施 DLQ：處理失敗請求
  4. 使用 Layers：重用相依性
  5. 監控與告警：設定 CloudWatch 警示
  6. 優化冷啟動：使用 Provisioned Concurrency
  7. 考慮使用 ARM64：提高性價比
  8. 使用 Lambda SnapStart：Java 函數冷啟動優化

Lambda 架構深入解析

核心概念

  • Function as a Service（FaaS）
  • 函數在運行環境中執行
  • 短期、無狀態
  • 事件驅動調用

Lambda 函數組成

組件說明
執行時期支援多種語言（Python、Node 等）
程式碼您編寫的函數程式碼
相依套件函數需要的套件
環境變數運行時配置

計費模式

  • 請求：每次調用計費
  • 執行時間：以 1ms 為單位
  • 記憶體分配影響價格
  • 免費層：每月 100 萬次請求 + 40 萬 GB-秒

Lambda 網路詳解

公共網路（預設）

  • 在 AWS 公共網路空間運行
  • 可存取公共 AWS 服務和網際網路
  • 無法存取 VPC 資源

VPC 網路

  • Lambda 在 VPC 子網路中運行
  • 遵循 VPC 網路規則
  • 可存取私有資源
  • 需要 NAT Gateway 存取網際網路

VPC 整合架構

Lambda → ENI → VPC 子網路 → 私有資源
                    ↓
              NAT Gateway → 網際網路

重要考量

  • ENI 建立需要時間（冷啟動因素）
  • 需要足夠的子網路 IP 地址
  • 使用 VPC Endpoint 存取 AWS 服務

Lambda 安全性

執行角色

  • Lambda 假設的 IAM 角色
  • 提供存取 AWS 服務的臨時憑證
  • 信任政策允許 Lambda 服務假設角色

資源政策

  • 控制誰可以調用 Lambda
  • 類似 S3 Bucket 政策
  • 可允許其他帳號或服務調用

調用模式

同步調用

  • 客戶端等待結果
  • 成功或失敗直接返回
  • 例如：API Gateway、CLI

非同步調用

  • 不等待結果
  • Lambda 自動重試（0、1、2 次）
  • 可配置 Dead Letter Queue
  • 例如：S3 事件、SNS

事件來源映射

  • 用於串流服務（Kinesis、DynamoDB Streams）
  • 用於佇列服務（SQS）
  • Lambda 輪詢來源

事件批次處理

設定說明
Batch Size每次調用的記錄數
Batch Window最長等待時間
部分批次失敗僅重新處理失敗的項目

Lambda 版本與別名詳解

版本

  • Lambda 函數可發布版本
  • 版本是程式碼和配置的不可變快照
  • $LATEST 指向最新未發布版本
  • 發布後無法修改

別名

  • 版本的指標
  • 例如：PROD 指向 version 1
  • 別名是可變的
  • 支援加權流量分配（藍綠部署）

部署策略

策略說明
全量部署直接切換到新版本
金絲雀小流量測試新版本
線性逐步增加新版本流量

執行環境生命週期

冷啟動 vs 暖啟動

類型說明時間
冷啟動建立新執行環境100ms - 數秒
暖啟動重用現有環境毫秒級

生命週期階段

INIT（冷啟動）→ INVOKE → SHUTDOWN

最佳化策略

  1. 程式碼放在 Handler 外部：利用環境重用
  2. 資料庫連線重用：避免每次建立連線
  3. 使用 Provisioned Concurrency：預熱執行環境
  4. 減少部署包大小：加快初始化

Lambda Destinations

概念

  • 非同步調用的結果路由
  • 成功和失敗可發送到不同目的地

支援的目的地

目的地類型說明
SNS Topic通知其他服務
SQS Queue佇列處理
Lambda 函數鏈式處理
EventBridge事件路由

Lambda 與其他服務整合

API Gateway + Lambda

客戶端 → API Gateway → Lambda → DynamoDB
  • 完全無伺服器架構
  • 按使用付費
  • 自動擴展

S3 + Lambda

上傳檔案 → S3 事件 → Lambda → 處理（壓縮、縮圖等）

Step Functions + Lambda

  • 編排多個 Lambda 函數
  • 狀態機工作流程
  • 錯誤處理和重試

Lambda 效能調校

記憶體與 CPU

  • 記憶體範圍：128MB - 10,240MB
  • CPU 與記憶體成比例分配
  • 1,769MB = 1 個完整 vCPU

效能建議

場景建議
CPU 密集型增加記憶體
I/O 密集型增加記憶體可能無幫助
高並發使用 Provisioned Concurrency
長時間執行考慮 Step Functions

Lambda 限制詳解

並發限制

類型限制
區域總並發1,000（可增加）
保留並發可為函數保留特定並發
Provisioned Concurrency預熱的執行環境數量

部署限制

項目限制
部署包（壓縮）50MB
部署包（解壓）250MB
容器映像10GB
/tmp 空間512MB - 10GB

Lambda 與 CloudWatch

自動整合

  • 所有 Lambda 調用自動記錄到 CloudWatch Logs
  • 每個函數有自己的 Log Group
  • 每個執行環境有自己的 Log Stream
