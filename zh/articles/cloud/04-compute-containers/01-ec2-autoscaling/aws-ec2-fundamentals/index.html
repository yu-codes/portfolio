概述

  • IaaS 模型的虛擬機器服務，可在雲端部署任意應用
  • 使用 Nitro 架構可達到最高 64,000 EBS IOPS，非 Nitro 僅 32,000
  • EC2 重新啟動時公網 IP 可能改變，若需固定 IP，使用 Elastic IP（每區預設上限 5 個）
  • 可透過 http://169.254.169.254/latest/meta-data 查詢 EC2 metadata
  • 建議同 AZ 部署多台 EC2 可節省區域間資料傳輸費用
  • 可用 VM Import/Export 匯入本地虛擬機映像檔轉為 EC2 AMI 使用

EC2 Hibernate

  • 可將 RAM 內容存至 EBS
  • 開機時載入，達成快速啟動
  • 適用於長時間運行的應用程式

EC2 實例類型分類

分類用途例子
T, M通用型Web 伺服器、小型資料庫、開發環境
C運算最佳化HPC、高效能批次處理、CPU ML
R, X, Z記憶體最佳化高效能資料庫、即時大數據分析
F, G, P加速運算GPU 密集應用、語音辨識、ML
D, H, I儲存最佳化高 I/O 作業如 Kafka、HDFS、ElasticSearch

EC2 啟動類型

類型說明適用場景
On-Demand按量計費，靈活但成本高短期、彈性需求
Reserved預付租用 1~3 年穩定、可預測工作負載
Scheduled Reserved指定週期性時段預約使用固定時段需求
Spot Instance最高折扣 90%，可容忍中斷批次處理、容錯應用
Dedicated Instance實體隔離，同租戶獨享硬體合規需求
Dedicated Host實體主機層級控管授權限制軟體

Reserved Instance 類型

  • Standard：可轉售
  • Convertible：可轉換型態

Spot Instance 進階

  • Spot Block：保留固定時段不中斷
  • Spot Fleet：組合 Spot + On-demand 依目標數量啟動

實例配額

  • Reserved：20 台
  • On-demand：1152 vCPU
  • Spot：1440 vCPU
  • 可提出申請調升

EC2 網路強化功能

類型說明最高速度
ENI (Elastic Network Interface)虛擬網卡，可附掛至 EC2-
ENA (Elastic Network Adapter)支援 C4、D2、M4 等類型100 Gbps
EFA (Elastic Fabric Adapter)OS-bypass，HPC/ML 低延遲-
Intel 82599 VF早期虛擬網卡10 Gbps

ENI 特性

  • 主要 ENI 不可移除
  • 次要 ENI 可 detach 並轉移至備援 EC2（Failover）

EC2 佈局策略（Placement Groups）

只能跨 AZ，不可跨 Region。

類型特性適用場景
Cluster同 AZ、同機架HPC、低延遲應用
Spread跨 AZ、跨機架，單 AZ 限 7 台高可用性需求
Partition同或跨 AZ，每 AZ 最多 7 分區Kafka、Cassandra

AMI（Amazon Machine Image）

  • 包含 OS、應用與設定的映像檔
  • 可用於快速建立新 EC2 實例
  • 可跨區複製，達成多區部署一致性

AMI 類型

類型說明
Public AMIAWS 或社群提供
Private AMI自建私有 AMI
Marketplace AMI第三方廠商提供

AWS Nitro System

  • 所有現代 EC2 實例的底層虛擬化平台
  • 提供更高效能、更好的安全性和更低的延遲
  • 支援最高 64,000 EBS IOPS 和 100 Gbps 網路

Elastic Fabric Adapter (EFA)

  • 用於 HPC 和 ML 工作負載的低延遲網路介面
  • 支援 OS-bypass 技術，降低延遲
  • 支援 MPI（Message Passing Interface）

EC2 Capacity Blocks for ML

  • 預留 GPU 運算容量用於 ML 工作負載
  • 可預約特定時間段的 GPU 實例

進階定價模式

Savings Plans（節省計畫）

類型折扣彈性
Compute Savings Plans最高 66%跨 EC2、Fargate、Lambda
EC2 Instance Savings Plans最高 72%特定實例系列和區域

Capacity Reservations（容量預約）

  • 確保在特定 AZ 中有可用的 EC2 容量
  • 無需長期承諾
  • 可與 Reserved Instances 或 Savings Plans 結合

最佳實踐

  1. 選擇適當實例類型：根據工作負載特性選擇
  2. 使用 Reserved Instance：穩定負載可節省成本
  3. 利用 Spot Instance：容錯應用可大幅降低成本
  4. 配置 Placement Group：根據需求選擇適當策略
  5. 建立 AMI：標準化部署流程
  6. 使用 Graviton 實例：ARM 架構可提供更好的性價比
  7. 啟用詳細監控：使用 CloudWatch 監控效能
  8. 使用 Instance Metadata Service v2 (IMDSv2)：提高安全性

虛擬化基礎

虛擬化概念

  • 虛擬化：在單一實體硬體上運行多個作業系統
  • Kernel：作業系統中唯一能直接與硬體互動的部分

虛擬化演進

類型說明效能
軟體模擬虛擬化軟體模擬硬體，OS 認為在真實硬體上運行慢
半虛擬化修改 OS 原始碼直接呼叫 Hypervisor較快
硬體輔助虛擬化硬體本身支援虛擬化快
SR-IOV網卡呈現多個邏輯裝置，減少主機 CPU 使用最快

在 EC2 中，SR-IOV 就是 Enhanced Networking

EC2 架構與韌性

基本架構

  • EC2 實例是虛擬機器（OS + 資源）
  • 運行在 EC2 Hosts 上
  • 共享主機（預設）或專用主機
  • Hosts = 1 個 AZ，AZ 故障 → 主機故障 → 實例故障
  • AZ 韌性：非常依賴所在 AZ

EC2 適用場景

  • 傳統 OS + 應用程式運算
  • 長時間運行的運算
  • 伺服器型應用（爆發或穩定負載）
  • 單體式應用堆疊
  • 遷移或災難復原
  • 傾向作為 AWS 預設運算服務

EC2 實例類型詳解

資源特性

  • 原始 CPU、記憶體、本地儲存容量和類型
  • 資源比例
  • 儲存和網路頻寬
  • 系統架構/供應商（ARM vs x86）
  • 額外功能（GPU、FPGA）

五大類別

類別特點適用場景
通用型均衡資源比例多樣化工作負載
運算優化CPU 優先媒體處理、HPC、ML、遊戲
記憶體優化記憶體優先大型記憶體資料集、資料庫
加速運算GPU/FPGA硬體 GPU、機器學習
儲存優化高 I/O資料倉儲、ElasticSearch

實例類型解碼

R5dn.8xlarge
- R：實例系列
- 5：世代
- dn：d=NVMe 儲存、n=網路優化
- 8xlarge：實例大小

購買選項深入解析

On-Demand

  • 預設選項
  • 不同大小的實例共享 EC2 主機
  • 按秒計費（運行時）
  • 相關資源（如儲存）無論實例狀態都會計費
  • 無中斷、可預測定價、無前期成本
  • 適用於短期、未知工作負載

Spot

  • AWS 出售未使用的 EC2 主機容量，最高 90% 折扣
  • 價格超過限制時實例會被終止
  • 永遠不要用於無法容忍中斷的工作負載
  • 適用於：非時間緊迫、可重跑、爆發容量、成本敏感、無狀態

Reserved

  • 長期一致使用
  • 匹配實例可享折扣或免按秒費用
  • 未使用的預約仍會計費
  • 1 年或 3 年期限
付款方式說明
No-Upfront同意期限即有折扣，按秒付費
All-Upfront全額預付，無按秒費用
Partial-Upfront部分預付，降低按秒費用

Dedicated Instance

  • 不與其他客戶共享硬體
  • 不擁有主機
  • 實例額外收費，專用硬體
  • 不管理容量

Dedicated Host

  • 主機完全分配給您
  • 支付主機費用，無實例費用
  • 需自行管理容量和資源
  • 適用於基於 Socket/Core 的授權需求
  • Host Affinity 將實例綁定到主機

Reserved Instance 進階

Scheduled Reserved Instances

  • 適用於長期使用但非持續運行
  • 選項範例：
    • 每天批次處理 5 小時
    • 每週五數據分析 24 小時
  • 最小 1200 小時/年，1 年期限

Capacity Reservations

  • 區域預約：該區域任何 AZ 享有折扣，但不預留 AZ 容量
  • 可用區預約：特定 AZ 的折扣和容量預留
  • 隨需容量預約：確保 AZ 中有容量，但以原價計費

EC2 Savings Plan

  • 按小時承諾，1-3 年期限
  • 通用運算金額或特定 EC2 Savings Plan
  • 涵蓋 EC2、Fargate、Lambda
  • 超出承諾部分以隨需價格計費

實例狀態檢查與自動復原

狀態檢查

每個 EC2 實例有 2 個狀態檢查：

檢查問題類型
系統狀態電源損失、網路中斷、主機軟硬體問題
實例狀態檔案系統損壞、網路設定錯誤、OS Kernel 問題

Termination Protection

  • 防止意外終止的屬性
  • 實現角色分離：初級管理員只能終止未保護的實例

水平與垂直擴展

垂直擴展（Vertical Scaling）

  • 調整 EC2 實例大小
  • 每次調整需重新啟動（中斷）
  • 較大實例通常有額外成本
  • 有效能上限
  • 不需修改應用程式
  • 適用於所有應用（包括單體式）

水平擴展（Horizontal Scaling）

  • 增加更多實例處理負載
  • 需要負載平衡器分配流量
  • 需要應用程式支援或外部 Session（無狀態）
  • 無停機擴展
  • 可無中斷移動連線
  • 通常更經濟，更精細的控制

Instance Metadata

概述

  • EC2 服務提供給實例的資料
  • 所有實例都可存取
  • 位址：http://169.254.169.254/latest/meta-data/
  • ⚠️ 請記住這個位址

可查詢內容

  • 網路資訊
  • 認證資訊
  • User Data
  • 未加密且未認證，視為可能曝露的資料

AMI 詳解

概述

  • AMI 用於啟動 EC2 實例
  • 由 AWS 或社群提供
  • Marketplace（可含商業軟體）
  • 區域性，有唯一 ID
  • 權限設定（公開、自己帳號、特定帳號）
  • AMI 是包含快照參考的容器

AMI 生命週期

  1. 啟動實例
  2. 設定實例
  3. 建立 AMI
  4. 從 AMI 啟動新實例

重要概念

  • AMI = 單一區域，只在該區域有效
  • AMI Baking：從設定好的實例建立 AMI
  • AMI 無法編輯，需建立新 AMI
  • 可跨區域複製（含快照）
  • 預設權限 = 您的帳號

EC2 Bootstrapping（User Data）

概述

  • 允許系統自我設定
  • 實現 EC2 建置自動化
  • 位址：http://169.254.169.254/latest/user-data
  • User Data 中的內容由實例 OS 執行
  • 僅在啟動時執行一次
  • EC2 不解讀，OS 需理解 User Data

重點

  • 對 EC2 來說是不透明的資料區塊
  • 不安全：不要用於密碼或長期憑證
  • 限制 16 KB 大小
  • 實例停止時可修改
  • 僅在啟動時執行一次

EC2 Instance Role 與 Instance Profile

概述

  • 讓 EC2 上的應用程式能存取 AWS 資源
  • 提供短期臨時憑證
  • 透過實例 Metadata 取得憑證
  • EC2 和 STS 確保憑證永不過期

重點

  • 憑證在 metadata 中
  • 路徑：iam/security-credentials/role-name
  • 自動輪換，永遠有效
  • 應永遠使用 Role 而非將 Access Key 放入實例
  • CLI 工具會自動使用 Role 憑證

Placement Groups 詳解

Cluster Placement Groups

  • EC2 中最高效能
  • 單一 AZ，同機架（有時同主機）
  • 所有成員直接連接
  • 最高 10Gbps 單一串流，一般 5Gbps
  • 最低延遲和最高 PPS
  • 權衡：幾乎無韌性
  • 建議使用相同實例類型，同時啟動

Spread Placement Groups

  • 保持實例分散
  • 可跨多個 AZ
  • 獨立機架，故障隔離
  • 每 AZ 限 7 個實例（硬限制）
  • 適用於需要保持分離的少量關鍵實例

Partition Placement Groups

  • 跨多個 AZ
  • 分為多個分區，每 AZ 最多 7 個
  • 每個分區有自己的機架，分區間不共享
  • 可指定實例放置位置或自動放置
  • 適用於拓撲感知應用（HDFS、HBase、Cassandra）

Enhanced Networking 與 EBS Optimized

Enhanced Networking

  • 使用 SR-IOV：NIC 支援虛擬化
  • 主機每張實體網卡有多個邏輯卡
  • 更高 I/O、更低主機 CPU 使用
  • 更多頻寬、更高 PPS、更低延遲
  • 大多數實例預設啟用或免費提供

EBS Optimized

  • EBS = 透過網路的區塊儲存
  • 歷史上網路是共享的（資料和 EBS）
  • EBS Optimized 表示 EBS 專用頻寬
  • 大多數實例預設支援並啟用
