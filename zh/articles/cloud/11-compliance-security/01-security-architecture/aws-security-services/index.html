AWS Shield

  • AWS 託管的 DDoS 防禦服務（L3 / L4 層）
版本說明保護範圍
Shield Standard免費CloudFront / Route 53
Shield Advanced付費EC2、ALB、CloudFront、Route 53

Shield Advanced 額外功能

  • 24/7 DDoS 回應團隊（DRT）支援
  • 成本保護（DDoS 攻擊期間的額外費用補償）
  • 進階報告和診斷

AWS WAF

  • Web 應用防火牆，保護 Layer 7（HTTP）流量
  • 可阻擋 SQL Injection / XSS 等常見攻擊
  • 可部署在 CloudFront、ALB、API Gateway、AppSync

WAF 規則類型

類型說明
IP 規則允許或阻擋特定 IP
地理位置規則依國家/地區過濾
速率限制規則限制請求頻率
AWS 管理規則預設安全規則集
自訂規則自定義過濾條件

AWS Firewall Manager

  • 集中式防火牆管理服務，跨帳號設定 WAF / Shield Advanced / Route 53 DNS 防火牆
  • 適用於：合規需求（如封鎖特定國家 IP）
  • 需要 AWS Organizations 支援

AWS GuardDuty

  • 威脅偵測服務，基於機器學習與異常行為判斷
  • 來源：VPC Flow Logs / DNS Logs / CloudTrail
  • 可偵測加密貨幣礦工活動等攻擊

偵測類型

  • 未授權存取
  • 加密貨幣挖礦
  • 惡意 IP 通訊
  • 資料外洩
  • 帳號洩露

Amazon Inspector

  • EC2 安全性評估服務（需要安裝 agent）
  • 覆蓋項目：
    • Network Reachability（檢查是否可被不預期存取）
    • Host Assessment（檢查漏洞與錯誤設定，對應 CVE / CIS 規範）

評估類型

類型說明
網路可達性檢查網路設定安全性
主機評估檢查 OS 漏洞和設定
CVE 評估對應已知漏洞資料庫
CIS 基準對應安全設定最佳實踐

Amazon Macie

  • 偵測與保護 S3 中的 敏感資料（如 PII） 的託管服務
  • 可主動識別資料類型並發出警報

可偵測的敏感資料類型

  • 個人識別資訊（PII）
  • 金融資訊（信用卡號）
  • 健康資訊（PHI）
  • 認證資訊（API 金鑰、密碼）

AWS Config

  • 多區帳號的 資源設定監控與稽核服務
  • 可整合 SNS 通知變更事件
  • 結合 CloudTrail 提供設定歷史紀錄
  • 適用於法規遵循需求（如 PCI-DSS、HIPAA）

Config Rules

  • 評估資源設定是否符合規範
  • 支援 AWS 管理規則和自訂規則
  • 可自動修復不合規資源

AWS Security Hub

概述

  • 集中式安全態勢管理服務
  • 彙整來自多個安全服務的發現
  • 跨帳號和跨區域檢視

合規標準

標準說明
PCI DSS v4.0支付卡產業資料安全標準
AWS Foundational Security Best PracticesAWS 安全最佳實踐
CIS AWS Foundations BenchmarkCIS 安全基準
NIST SP 800-53美國國家標準
ISO 27001資訊安全管理系統

主要功能

  • 自動化發現關聯：彙整來自 GuardDuty、Inspector、Macie、IAM Access Analyzer 的發現
  • 安全分數：量化安全態勢的合規狀態
  • 自訂洞察：建立自訂查詢識別特定安全問題
  • 自動化修復：整合 Systems Manager Automation 和 Lambda

AWS IAM Access Analyzer

  • 識別與外部實體共享的資源
  • 分析 IAM 政策以發現過度權限
  • 支援 S3 存儲桶、IAM 角色、KMS 金鑰等資源
  • 跨帳號分析支援 AWS Organizations

最佳實踐

  1. 啟用 GuardDuty：持續監控威脅
  2. 使用 Config Rules：確保資源合規
  3. 定期執行 Inspector：掃描漏洞
  4. 保護敏感資料：使用 Macie 掃描 S3
  5. 分層防護：同時使用 Shield 和 WAF
  6. 使用 Security Hub：集中管理安全發現
  7. 啟用 IAM Access Analyzer：識別意外的存取權限
